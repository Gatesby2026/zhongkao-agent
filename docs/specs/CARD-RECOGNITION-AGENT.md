# 答题卡识别 Agent — 设计方案（讨论稿 v0.2）

> **日期**：2026-06-07
> **状态**：🟢 选择题"像素回投"探针已在 3 张真卡实测验证（见 §4.1.1）；下一步识别过程界面可视化（见 §11）
> **关联**：[`CHOICE-DETECTION-METHOD.md`](CHOICE-DETECTION-METHOD.md)（Method D / Path A-D 历史与实测）·
>   `scripts/answer-card-ocr/pixel_probe.py`（像素探针原型）· `scripts/answer-card-ocr/detect.py`（现有识别）·
>   `server/tasks.py`（流水线 + need_manual 门禁）
> **背景痛点**：答题卡识别是老大难。实测线上旧识别在英语密集涂卡网格上崩盘（郭嘉瑞顺义英语
>   33 题误判全失 → 15.5/60），单模型 confidence 不可信（"自信地错"，0.85-0.95 仍错）。
>   → 用 **agent 方式**：质检模块 + 循环控制 + 逐步反思改进，直到识别出合格结果，否则兜底人工。

---

## 0. 范围（已锁定）

**只解决"识别准不准"，不碰判分。**
- 产出 = 学生**实际涂/写了什么**（带逐题置信度），不是"正确答案"。
- 结构化数据（题目/答案/评分要点）**只当质检的尺子，不产出分数**。判分是下游另一条线。

---

## 1. 最关键原则：识别必须对"标准答案"盲

既然产出是"学生实际作答"，识别就**不能朝答案 key 收敛**：
- 选择题质检**绝不拿标准答案当靶子**——否则学生涂了 B（错），识别器为"匹配标准答案 A"把它纠成 A，识别被污染 = 作弊。
- 主观题同理：学生可能答错/跑题，**不能用标准答案验"读得对不对"**。

**铁律：结构化数据只能验"结构/位置/合理性/格式"，绝不验"对错"。** critic 的输入 =
**图像信号 + 卷子结构（题数/题型/题号/材料主题/答案格式），永不含标准答案。**

---

## 2. 为什么 agent 循环在这能成（以及边界）

循环改进的本质是**朝可测量的目标逼近**；critic 不可靠则循环瞎转。
- **选择题**：有一批**物理上可验证的硬信号**（单选不可能涂俩、四选不可能全涂、题数应等于卷子、答案只能 A-D、多方法可交叉印证）→ 循环**真能收敛**。这是值得做的根本原因。
- **主观题**：识别产出其实只是"把每题作答区**裁对**"（报告展示裁图，不展示 OCR 文本）。结构化数据验"裁的位置/对齐"，不验答案对错。

---

## 3. 五个模块

```
            ┌─────────── 控制器（循环 + 预算）───────────┐
答题卡 ──▶ 感知/工具层 ──▶ 候选结果 ──▶ 质检/批判 ──▶ 合格?
              ▲                            │           │是→输出（逐题置信度）
              │                            ▼否
              └──── 反思器（缺陷→针对性动作）◀──────────┘
                                           │预算耗尽且仍有残留
                                           ▼
                                  兜底：残留题转人工（need_manual，已建好）
```

| 模块 | 职责 |
|---|---|
| **① 感知/工具层** | 定位（腾讯字母-bbox 最稳 / 粗裁兜底 / 旋转纠正）· 读取（像素 blob / 腾讯缺字母 / VLM 单选约束读：整带·逐行·**逐题放大**）· 图像操作（裁/放大/旋转/二值化）· **像素探针**（在声称涂了的格子做黑度检测） |
| **② 质检/批判** | 整套的心脏，见 §4。逐题输出 `{answer, confidence, 缺陷标签}`，**对答案 key 盲** |
| **③ 控制器** | 跑循环、管预算（最多 N 轮 / 花多少钱封顶）、决定收敛或兜底 |
| **④ 反思器** | 拿质检的**具体缺陷**诊断，选**不同的**下一步（非盲目重试），见 §5 |
| **⑤ 兜底** | 即已建的 `need_manual`：**只把搞不定的残留题**交人工，预填最佳猜测，负担最小化 |

## 4. 质检/批判模块（心脏）

### 4.1 选择题：4 层信号（便宜→贵），全部图像/结构派生，对 key 盲

| 层 | 信号 | 成本 | 抓什么 |
|---|---|---|---|
| **L1 物理/结构硬约束** | 单选=1字母 / 答案∈{A-D} / 题数==卷子期望 / 无四选全涂 / 覆盖率 | 零、确定性 | 郭嘉瑞那种崩盘一票否决 |
| **L2 多方法共识** | 跑 2-3 个独立识别器，**一致=可信，分歧=不确定集** | 中 | 定位"哪几题要重点攻" |
| **L3 多视角一致性** | 同格子在不同裁切/旋转/放大下读，是否一致 | 中 | 边界涂卡、倾斜误读 |
| **L4 回投像素核验** | 在"声称涂 A"的格子做黑度探测——真黑才算；顺便看别格是否真没涂 | 低 | 抓"自信地错"（VLM 高 conf 但实际没涂） |

> 核心洞察：**别信单模型 confidence（它会自信地错），信"多源印证 + 物理约束 + 回投核验"。**
> 这是跟"再调一遍 prompt"的本质区别。
> 已落地的雏形：`detect.choice_reliability()`（L1 部分：多字母/四选全涂/覆盖率/大面积空 → need_manual）。

### 4.1.1 像素探针实测（2026-06-07，3 张真卡，已验证）

`scripts/answer-card-ocr/pixel_probe.py`：腾讯定位"没涂字母"bbox → 用**字母身份 + 步距反推**每题
A-D 四格坐标（含被涂黑、OCR 漏读那格）→ 量每格黑度 → 最黑=涂的，`margin=最黑-次黑`=置信度。
**完全 model-free，是 VLM 的独立证伪源。**

| 卡 | 结果 | 关键观察 |
|---|---|---|
| 贾小淇数学（干净，真值 ACCBCABD） | **7/8**，margin 0.19-0.28 | 唯一错的 Q5 正是"探针 B vs 缺字母 C"**分歧**那题；并**救回** Q4/Q8（缺字母漏读尾字母 BCD/CD，探针拿对 B/D） |
| 郭嘉瑞英语（线上崩盘卡） | **干净单字母**，margin 0.4-0.68 | 密度一格 ~0.5、其余 ~0.00；而线上 multi-merge 在此卡吐 AC/ABCD 垃圾、15.5/60 |
| 姜惟宁海淀数学（正常） | 单字母，margin 0.09-0.30 | 低 margin 的 Q4（0.087）+ 与缺字母分歧 → 正是该 flag 的不确定题 |

**验证到的结论（升级 L2/L4 设计）**：
1. **像素探针是强独立信源，便宜、model-free、不跟 VLM 一起犯错**——崩盘卡上它给干净结果，这是 multi-detector 投票做不到的（它们会一起失败）。
2. **margin = 置信度**：>0.2 稳；<0.1 模糊。但**单源 margin 不够**（贾 Q5 margin 0.24 仍错，涂卡溢出到邻格）——必须靠"探针 vs 读取器分歧"兜底。
3. **"探针 vs 读取器（缺字母/VLM）分歧" 精准圈出不确定集**（贾 Q5、姜 Q4、郭 Q19），其余高一致 → 高置信直接收。
4. **探针能反向救读取器短板**（缺字母漏尾字母时探针补全）。
5. 探针靠"OCR 字母位置 + 步距反推网格"——**定位（腾讯）即使在'读'失败时也能用**，几何稳定可得。

**🔑 超预期结论：像素探针好到可当"主识别器"，不只是验证器。** 干净卡上它独立就能出结果、比 VLM 便宜；
VLM 退成"只在不确定题上放大打平局"。**成本模型反过来**：主路便宜（像素），贵的（VLM）只花在 2-3 道难题。
→ §4.1 共识链定稿为：**像素探针（主） + VLM（打平局） + 两者分歧/低 margin → 不确定集 → 放大/兜底人工**。

**待硬化**：① 涂太满 / 可读字母太少 → 步距估歪，需护栏（每行 ≥N 可读字母才信几何）；② margin 阈值需更多卡标定。

### 4.2 主观题：识别 = 裁对区域；结构化数据验位置/对齐/格式（不验对错）

- **题号锚定**：裁图里印刷题号 == 期望题号（纯结构）
- **主题对齐**：裁图内容领域 ↔ **题目/材料**领域（注意是"题目"不是"标准答案"）。
  治 Q19/20/24/27 张冠李戴：用"这块内容像哪道题的材料"来**反向归位裁图到题号**。
- **格式期望（软信号，验证型题）**：甲乙丙丁=4选1、默写=一句诗、数学填空=一个值——
  读出不符**格式**说明裁错/读崩。注意验的是**格式**，不是"读出值==标准答案"。
- OCR 读不出乱手写的硬限仍在：读出≈乱码/空 → 信号是"裁错或读失败 → 换裁/放大重读，
  实在不行**只展示裁图给人看/确认**"。

## 5. 反思器：缺陷 → 针对性动作（非盲目重试）

| 质检缺陷 | 诊断 | 下一步（换策略） |
|---|---|---|
| 覆盖率低/定位失败 | 没框对涂卡区 | 换旋转角 / 换定位器 / 让 VLM 给区域再裁 |
| 单选读出多字母 | 那题裁糊/涂卡溢出 | **只放大这一题**重读 + 像素探针定夺 |
| 两方法在 Qx 分歧 | 单题歧义 | 放大 Qx + 第三方法/像素探针**打破平局** |
| 全卷读出同一字母 | 网格 base_x 系统性错位 | 重标定网格基准 + 重投 |
| VLM 反复给 ABCD | 模型这格不可靠 | **弃用 VLM，改纯像素黑度**比四格 |
| 主观题裁图主题不符 | 裁错/错位 | 按主题重新对齐裁图到题号 / 重裁 |

**核心：每轮缩小到"不确定集"，把算力砸难题上**（像读房诗尧：先读整带，只对 Q4/Q8 放大），不是每轮重读整卷。

---

## 6. 合格判据（Definition of Done）

一次合格识别 =
- **选择题**：每题一个 key 盲字母，过 L1 物理约束 + L2 多源共识 +（分歧/低置信时）L4 像素回投；
- **主观题**：每题裁到正确区域，过题号锚定 +（可选）主题对齐；
- **残留**（定不下来）→ 兜底人工：选择题录字母 / 主观题确认裁图。

全程不出现"正确答案"，不产出分数。

---

## 7. 架构选型

- **方案 A 全 LLM agent**（Claude 带视觉+工具自主 reasoning）：最灵活、难 case 上限高，但贵/慢/不可控/难复现。
- **方案 B 确定性控制器 + 可验证 critic + VLM 当工具**（**推荐**）：反思写成"缺陷→动作"策略表，
  VLM 只在"读某题/某区域"被调用。便宜、可预测、可复现、好调试，把"智能"用在刀刃（读难题）。
- **建议**：B 为主干 + 在"难题残留"那步**注入 VLM/Claude 推理**（看放大图自己定）。全自主 agent 留作 B 兜不住的最后一档。

---

## 8. 成本模型

便宜优先，只对不确定集升级：像素 blob（零）→ 腾讯（¥0.05）→ VLM 整带（几分）→ VLM/Claude 逐题放大（只花在 2-3 道难题）。预算硬封顶，到顶兜底人工。

## 11. 识别过程界面可视化 + 用户修订（设计建议）

> **动机（用户提）**：识别过程要能在界面上看到——① 方便开发分析定位问题；② 用户能看到识别对不对；
> ③ 不对的话，后边给用户机会自行修订补充。

### 11.1 核心设计：把"识别结果叠在用户自己的答题卡照片上"

最有说服力的可视化不是表格，而是**在用户上传的原图上叠标注**——用户一眼就能比对"系统读到的"
和"我实际涂的"是否一致。

```
┌──────────────────────────────────────────────┐
│  [用户答题卡原图]                               │
│   1. Ⓐ B  C  D     ← 绿圈：高置信，已自动采纳    │
│   2.  A  B Ⓒ D                                 │
│   5.  A 〔B〕Ⓒ D    ← 黄框：不确定（探针B/读取C分歧），请确认 │
│   ...                                          │
│  [✓ 全部正确]   [✎ 修正第5题]                   │
└──────────────────────────────────────────────┘
```

- **绿（confident）**：探针 + 读取器一致 + margin 够 → 自动采纳，仅供一瞥核对。
- **黄（uncertain）**：分歧 / 低 margin / 物理约束触发 → **高亮 + 要求用户确认**，是注意力焦点。
- **红（无法识别）**：定位失败 / 覆盖率过低 → 直接转手工录入。

### 11.2 两个受众，两层视图（同一份 trace 数据驱动）

| 视图 | 受众 | 内容 |
|---|---|---|
| **简版（默认）** | 家长/学生 | 原图叠绿/黄标注 + "黄题请确认" + 一键修正。低摩擦：好的瞄一眼，可疑的点一下改 |
| **详版（可展开/开发模式）** | 开发/排障 | 逐题证据面板：裁图小图 + 四格黑度条(A/B/C/D) + 探针答案 vs 读取器答案 + 共识判定 + 走过几轮/放大了哪题 |

详版本质就是把 `pixel_probe.py` 的 `dens(A/B/C/D)` + margin + ocr_missing + VLM 读数**逐题摊开**，
正是定位问题的最佳现场（哪格黑度多少、为什么判这字母、和谁分歧）。

### 11.3 交互/修订流程（复用已建后端）

1. 识别完 → 出**确认页**（不是直接出报告）。绿题默认勾选，黄题置顶待确认。
2. 用户在原图上**点正确的格子**即可改任一题（黄题预填"最佳猜测"，减负担）。
3. 提交 → 走已建的 `POST /api/analyses/{aid}/manual-choices`（source=manual, conf=1.0），再 `submit_resume` 判分出报告。
4. **门禁升级**：现在是"不可信才转人工"；新设计是"**始终给一次确认机会**"——但只对黄题**要求**动作，绿题一瞥即过。平衡透明与摩擦。

### 11.4 与 agent 循环的关系

- 界面展示的是 agent 收敛后的**终态** + 关键 trace（"Q5 没把握 → 放大重读 → 仍分歧 → 标黄请你定"）。
- 把"循环内的不确定集"直接变成"界面上的黄题"——agent 的反思过程对用户**可见且可接管**，
  这正好闭合 §6 合格判据里的"残留 → 兜底人工"。

### 11.5 落地建议（小步）

1. 先让 `pixel_probe.py` + 读取器输出**统一的逐题 trace JSON**（`{qid, final, conf, probe, reader, dens, status:green/yellow/red, reason}`）。
2. 后端把该 trace 落库随 analysis 存，新增 `GET /api/analyses/{aid}/recognition-trace`。
3. 前端先做**详版表格**（开发/排障用，最快见效）；再做**简版原图叠标注 + 点选修正**（用户用）。
4. 修正提交复用现成 `manual-choices` 接口，零新判分逻辑。
