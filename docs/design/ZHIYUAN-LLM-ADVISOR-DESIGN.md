# 志愿草表 · 双轨生成 + 大模型顾问设计

> 规则基线见 `ZHIYUAN-DRAFT-ALGORITHM.md`(六层规则 / 七不变量 / 阈值)。本文聚焦**第二轨:大模型付费方案**与**双轨协作**。
> 决策定档(2026-06,与用户商定):①LLM 改良规则稿(非从零生成) ②Opus 4.8 走 Anthropic **API**(非订阅) ③加孩子画像 ④一次性 **¥100** ⑤已开 P0。
> 第一原则(继承):**涉及孩子升学,绝不编造录取数据**。LLM 只对我们喂入的结构化数据做判断与表达,所有数字(线/位次/距离/名额)来自 KB。

---

## 1. 双轨架构

```
家长输入 ─▶ ① 规则引擎(免费·保底)  recommend.py + 前端 resetDraft
            确定性·秒级·可解释·永远兜底           → 规则版草表 + 模板化理由(人人可见)
                    │ 规则稿 + 候选校结构化数据 + 画像 作为脚手架
                    ▼
            ② 大模型引擎(付费 ¥100)  Opus 4.8 / Anthropic API
            复核 / 调整 / 讲理 / 个性化 / 情形预演    → 优化版草表 + 深度分析报告
            ★不重算,只对喂入数据做判断与表达
                    ▼
            ③ 规则校验闸门(复用规则引擎)
            合规?(身份/批次/铁保底/无编造校码) → 通过才展示;失败回落规则版
```

**为什么 LLM 基于规则稿改良,而不是从零生成**:
1. 永远有底——LLM 超时/降级,用户仍拿到规则版。
2. 抗幻觉——LLM 的活儿从"算"变"审+解释+个性化",数字全来自 KB。
3. 便宜——一次生成,不做检索/计算。
4. 可问责——LLM 可推翻规则稿,但必须给理由(这正是付费价值)。

## 2. 免费 vs 付费定位

| | 规则版(免费) | 大模型版(¥100 一次性) |
|---|---|---|
| 交付 | 一份能直接用的合规草表 + 每条一句模板理由 | 一份懂你家情况的方案 + 详细分析报告 |
| 回答 | "能上哪些" | "我家该怎么填、为什么、万一呢" |
| 类比 | 计算器 | 私人志愿规划师 |

账号体系已就绪(`server/auth`,手机/邮箱登录);收费即"免费=规则版,付费解锁=大模型版+报告"。

## 3. 大模型输入(喂什么)

1. **规则稿**:规则引擎产出的三批次草表 + 每条 band/理由/参考位次。
2. **候选校结构化数据**(KB,只喂 curated/verified;低可信标"待核"):三年录取线+位次、2026 朝阳口径预估(pred_2026_cy / cy_equiv / below_control)、梯队、特色 style/tags、高考出口 U 分+增值 residual、校额到校名额、统筹强度、高德距离、批次计划、新校估。
3. **家庭/孩子画像**(见 §5)。
4. **规则口径**(`ZHIYUAN-DRAFT-ALGORITHM.md` 摘要,作为事实基线写进 system prompt)。

**防幻觉硬约束**(写进 prompt + 后置校验):只能引用喂入的数字;不得发明学校/代码/线;缺数据说"待核";可调整规则稿排序但须给依据。

---

## 4. 大模型报告结构(付费价值)

付费交付 = 结构化报告,六段:
1. **个性化总判**(一段):基于位次/住址/身份/画像,整体冲还是稳、最大机会点、最大风险点。
2. **逐志愿理由 + 对规则稿的调整**:每个志愿为何在此位;哪几处相对规则版做了调换,理由。
3. **关键权衡**:通勤vs层次 / 住宿换更好校 / 理科强校vs均衡 / 大小年波动。
4. **情形预演(敏感性)**:超常+10分位次到X→怎么调;失常→保底够不够。(教研员级价值)
5. **跨批次组合策略**:校额到校/统筹/贯通如何与统招配合(家长最易错、最值钱)。
6. **行动清单 + 待核**:要打电话核的校区、要查的简章、低可信数据提示。

## 5. 孩子画像问卷(付费版差异化燃料)

免费版不问;付费版多问 5–6 题(可选,缺省不阻塞):
- 文理倾向 / 单双科优势(强弱科)
- 性格:自驱型 vs 需盯
- 住校意愿(强/弱/不接受)——影响远郊统筹取舍
- 目标:冲名校 / 求稳上岸 / 特长方向
- 国际/出国路线接受度(影响是否纳入民办国际)
- 风险偏好(已有 form.risk,可复用)

存入 `app_profiles`(app='zhiyuan',与现有 profile 同表),供 LLM 个性化。

## 6. 数据缺口(按价值排序)

已有:三年线+位次、2026 朝阳口径预估、梯队、特色、高考出口+增值、校额名额、统筹强度、距离、批次计划、新校估、民办/中职/贯通。

**关键缺口(LLM 方案的燃料,7 月起补)**:
1. **2026 出分后一分一段表**——判档生命线,现全是估。**最高优先(7/9 出分)**。
2. **2026 招生计划数**——扩招/缩招直接改位次,7 月初发布。
3. **校额到校"逐初中"真实名额表**——中低位次最值钱杠杆;现 `chaoyang_xeddx.yaml` 覆盖度待核。
4. **分专业/分班型录取线**(实验班vs普通班)——影响填哪个专业代码;现多为校级线。
5. **质性信息**(校风/师资/管理松紧/作息/分班/住宿条件)——LLM 判"适不适合你孩子"靠这些;`campus_life.json` 有部分,深度不足。
6. **画像问卷**(§5)——见上。

## 7. 模型与成本

- **模型**:Claude **Opus 4.8**(`claude-opus-4-8`)经 **Anthropic API**(独立 API key,**非 MAX 订阅**——订阅不可程序化对外服务)。说理强,报告质量是卖点。
- **定价参考**:输入 $5 / 输出 $25 每百万 token。单份报告 ≈ 输入 20–40K + 输出 ~5K ≈ **$0.28 ≈ ¥2**;开 prompt 缓存(KB/规则前缀缓存读 0.1×)更低。¥100 定价毛利 ~98%。
- **缓存**:KB 候选 + 规则口径作稳定前缀(`cache_control` ephemeral),画像/草表放后段。
- **失败回落**:LLM 超时/报错/校验不过 → 返回规则版,提示"AI 版稍后重试"(已付款不损失)。
- **开发**:用 MAX 套餐(Claude Code/claude.ai)打磨 prompt/报告;生产用 API key(拟进 systemd EnvironmentFile,同短信/邮件)。

## 8. 后置规则校验闸门

LLM 报告/草表展示前,复用规则引擎校验(也是卖点"AI 方案也过同一套合规校验"):
- 身份×批次合法(非京籍不进普高统招;往届无指标分配/贯通)
- 批次锁定不违反(②不填统招本可达)
- ③ 有铁保底(末位一定能上)
- 无编造:每个学校/代码/线能在 KB 命中
- 不过 → 丢弃 LLM 结果,回落规则版 + 记录 case
