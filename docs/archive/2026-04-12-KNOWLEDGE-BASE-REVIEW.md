## 五、数据来源与可信度评估

### 总体分布

| 类别 | 内容 | 可信度 | 占数据量 |
|------|------|--------|---------|
| 真实公开数据 | 数学真题分析、录取分数、课标、教材映射 | ⭐⭐⭐⭐⭐ | ~25% |
| 真实试卷转录 | 101 套数学模拟卷（2300+ 题） | ⭐⭐⭐⭐ | ~65% |
| LLM 生成 + 已标注 | 数学诊断标准、学习路径 | ⭐⭐⭐ | ~3% |
| LLM 生成 + 未标注 | **新增 4 科全部数据** + 数学易错点 | ⭐⭐ | ~7% |

### 新增科目的特殊风险

新增 4 科（语文/英语/物理/道法）的考纲和真题分析**全部由 LLM 生成**，未与 `knowledge-original/` 中的真题原卷交叉验证。这意味着：

1. **题型结构可能有误** — 已在英语科目验证了这一风险（两套矛盾的改革后结构）
2. **分值分配可能有误** — 已在语文科目验证（权重加和 105%）
3. **逐题分析可能是编造的** — LLM 在生成 2021-2025 逐年真题分析时，部分题目内容可能是"合理想象"而非基于真实试卷
4. **时政热点归因可能有误** — 道法科目的年度热点与命题的对应关系需要验证

> **核心判断：** 新增科目数据的**结构和框架**是高质量的（与数学模板一致），但**内容的准确性**是未验证的。这批数据适合用于「界面展示」和「粗粒度参考」，但**不适合直接驱动精准的学习规划推荐**，直到用真题原卷校准完毕。

## 七、产品级别的关键缺失（更新）

### 缺失 1：新科目数据未经验证（影响：高）

`knowledge-original/` 目录下已有语文/英语/物理/道法的真题原卷 PDF（2005-2025），但目前**完全没有使用**。应该用这些原卷校准 LLM 生成的逐题分析。

**建议做法：**
1. 选取 2025 年真题原卷，逐题对比 LLM 生成的分析文件
2. 修正题目内容、分值、知识点标签中的错误
3. 确认题型结构（特别是英语 2024 改革后的结构）

### ~~缺失 2：诊断能力仍仅限数学~~ → 已解决

✅ Phase 3 已为语文/英语/物理/道法建立完整的 L0-L3 诊断标准（19 个模块文件），每模块含：
- signals（各级别特征信号）
- diagnostic_questions（诊断例题 + 典型错误）
- exam_benchmarks（真题锚点）
- 配套学习路径（19 个文件，含 steps/resources/pass_criteria）

⏳ 待做：代码层适配加载新科目的 diagnostics 和 learning-paths 文件

### 缺失 3：全科总分规划（影响：中）

当前产品是「单科规划器」。对于 510 分制的北京中考，学生更需要的是「6 科总分最优分配」——比如数学已 90 分、语文才 60 分的学生，应该优先提语文。这需要：
- 各科目标分映射（类似数学的 math-target-mapping）
- 6 科时间分配优化算法
- 跨科目优先级排序

### 缺失 4：Prompt 全科注入（影响：中）→ 部分解决

✅ `prompt-builder.ts` 已新增 `buildMultiSubjectPrompt()`、`analyzeSubjects()`、`buildSubjectKBContext()` 函数，支持多科目知识库注入和 ROI 驱动的时间分配。
⏳ API 路由层尚未切换到新函数，功能未上线。
