# 中考智能学习规划 — 用户画像咨询报告

> 报告人：资深教育产品经理
> 日期：2026-04-12
> 产品版本：当前线上版（39.103.70.47:3002）

---

## 一、核心发现

本产品当前设计隐含了一个假设：**用户是学生本人，且能准确自评各模块水平**。但根据教育行业实际情况，这个假设存在重大偏差。

**关键结论**：
1. 真正的决策者和付费方是**家长**，而非学生
2. 初三学生无法准确自评 7 个模块的水平（"还行"到底是 L1 还是 L2？）
3. 产品缺少**持续粘性机制**——目前是"用一次就走"的工具
4. 三个最有价值的用户场景中，产品只覆盖了 1.5 个

## 二、用户角色矩阵

### 角色 1：焦虑型家长（核心付费方）

| 维度 | 描述 |
|------|------|
| **典型画像** | 35-45岁，北京城区，大专/本科以上，双职工家庭 |
| **核心焦虑** | 孩子成绩中等（65-80分），不知道能上什么学校，不知道怎么帮 |
| **行为特征** | 频繁搜索"中考数学提分"、"海淀一模分数线"、在家长群比较信息 |
| **使用场景** | 晚上 9-11 点孩子睡后，用手机/电脑研究升学信息 |
| **决策模式** | 先自己试用，觉得靠谱再让孩子用；会截图发到家长群 |
| **痛点** | ①不懂数学教学法，没法辅导 ②请家教太贵(300-500/时) ③孩子不配合 |
| **对产品的期望** | "告诉我孩子哪里差、怎么补、能提多少分" |
| **占比预估** | 60% 的首次访问来自家长 |

**当前产品缺口**：
- 没有"家长视角"入口——家长不关心 L0/L1 级别，关心"能不能上八十中"
- 表单要求填 7 个模块自评——家长填不了，学生不愿填
- 没有"分享给孩子"功能——家长看完规划想发给孩子执行
- 缺少"进度跟踪"——家长想每周看到变化

### 角色 2：自驱型学生（核心使用方）

| 维度 | 描述 |
|------|------|
| **典型画像** | 14-15 岁，初三，有目标学校，能自主安排学习时间 |
| **核心需求** | "我知道函数差，但不知道该从哪开始练" |
| **行为特征** | 会用手机/iPad 搜题、看讲解视频，有一定信息检索能力 |
| **使用场景** | 放学后/周末，坐在书桌前，旁边放着课本和教辅 |
| **决策模式** | 看到具体的每日计划才会行动，越具体越好 |
| **痛点** | ①学校作业多，自主时间有限(0.5-1.5h/天) ②练了但分没提上来 ③不知道先练什么 |
| **对产品的期望** | "直接告诉我今天做什么、做哪页哪道题" |
| **占比预估** | 30% 的首次访问 |

**当前产品匹配度**：专项突破模式（/drill）较好匹配，但"全面规划"表单过重。

### 角色 3：被动型学生（最大沉默群体）

| 维度 | 描述 |
|------|------|
| **典型画像** | 14-15 岁，初三，成绩中下（50-70 分），被家长要求使用 |
| **核心状态** | 不知道自己差在哪，也不太关心；做题凭感觉 |
| **行为特征** | 家长发的链接可能不打开；打开后看到长表单直接关掉 |
| **使用场景** | 被家长盯着打开产品，快速填完了事 |
| **痛点** | ①看不懂规划里的术语 ②计划太多执行不完 ③没有正反馈，很快放弃 |
| **对产品的期望** | "越简单越好，别让我填太多东西" |
| **占比预估** | 这个群体不会主动来，但是家长群体的"下游用户" |

**当前产品致命缺口**：
- 7 个模块自评 × 5 个选项 = 35 次点击——这类学生会直接离开
- 没有"最小可用"入口（比如：只输入分数就给建议）
- 缺少激励机制

### 角色 4：课外辅导老师/机构

| 维度 | 描述 |
|------|------|
| **典型画像** | 25-40 岁，一对一/小班教学，带 5-20 个初三学生 |
| **核心需求** | 快速了解每个学生的水平分布，批量生成学习计划 |
| **使用场景** | 备课时使用，或展示给家长看以增加信任感 |
| **痛点** | ①每个学生都要手动分析很耗时 ②缺少数据支撑，说服力不够 |
| **对产品的期望** | "帮我 5 分钟出一份专业的诊断报告给家长看" |
| **占比预估** | 10%，但 LTV 最高（可能批量使用） |

**当前产品缺口**：没有批量/多学生管理功能，没有 PDF 导出。

## 三、用户旅程分析

### 3.1 当前用户旅程（Plan 模式）

```
触达 → 首页 → 选"全面规划" → 填表单(~3min) → 等待(~10s) → 看报告 → 离开
 │                                 │                              │
 │  ⚠️ 60%家长在这里犹豫           │  ⚠️ 30%学生在这里放弃         │  ⚠️ 看完就走
 │  "这是给谁用的？"               │  "7个模块我怎么评？"          │  没有后续
```

### 3.2 当前用户旅程（Drill 模式）

```
触达 → 首页 → 选"专项突破" → 4步表单(~30s) → 看计划 → 离开
                                                        │
                                               ⚠️ 同样没有后续
```

### 3.3 理想用户旅程

```
触达 → 角色选择 → 极简输入 → 看到"钩子结果" → 深入填写 → 完整报告 → 每周回访
 │     (家长/学生)  (1个数字)  (目标校匹配度)   (可选)      (付费墙?)    (推送)
 │
 ├→ 家长路径：输入分数+区 → 看到"可冲/稳妥/保底"学校 → 想看详细规划 → 让孩子填自评
 ├→ 学生路径：输入分数 → 看到最需要补的模块 → 选一个开始练 → 每天打卡
 └→ 老师路径：批量输入学生信息 → 导出诊断报告 PDF
```

## 四、核心场景优先级

| 优先级 | 场景 | 用户 | 频次 | 当前覆盖 |
|--------|------|------|------|----------|
| **P0** | "孩子考了 72 分，能上什么学校？" | 家长 | 高频（每次考试后） | ❌ 未覆盖 |
| **P0** | "函数不行，给我出个练习计划" | 学生 | 中频（每周1-2次） | ✅ /drill 覆盖 |
| **P1** | "全面诊断+整体规划" | 学生+家长 | 低频（学期初/模考后） | ✅ /plan 覆盖 |
| **P1** | "上周计划执行得怎么样？" | 学生 | 高频（每周） | ❌ 未覆盖 |
| **P2** | "给家长看个专业报告" | 老师 | 中频 | ❌ 未覆盖 |
| **P2** | "今天做了 10 道题，对了 7 道" | 学生 | 高频（每天） | ❌ 未覆盖 |
| **P3** | "模考真题在线练+自动批改" | 学生 | 中频 | ❌ 未覆盖 |

## 五、用户细分 × 产品策略矩阵

### 5.1 按成绩分层

| 分层 | 分数区间 | 心态 | 核心诉求 | 产品策略 |
|------|----------|------|----------|----------|
| **挣扎层** | 40-60 分 | 迷茫/放弃边缘 | "哪怕及格就行" | 极简计划，只抓基础题（选择1-5 + 填空9-12），每天 30 分钟，正反馈优先 |
| **中等层** | 60-80 分 | 有希望但不确定 | "能不能上区重点？" | 薄弱模块突破 + 目标校对标，是产品**最大目标群体** |
| **良好层** | 80-90 分 | 有目标想冲刺 | "再提 5 分上十一学校" | 精准补漏 + 压轴题训练，时间紧迫感强 |
| **拔尖层** | 90+ 分 | 求稳怕失误 | "别丢冤枉分" | 易错点巩固 + 考试策略，需求频次低 |

**当前产品问题**：四层用户拿到的规划结构是一样的。挣扎层不需要"压轴题分析"，拔尖层不需要"基础概念回顾"。

### 5.2 按时间阶段

| 阶段 | 距中考 | 用户状态 | 产品侧重 |
|------|--------|----------|----------|
| **长线期** | 6+ 月 | 较从容，愿意系统学 | 全面诊断 + 分模块提升路线 |
| **冲刺期** | 2-6 月 | 焦虑上升，要看到效果 | 重点模块突破 + 每周可见进步 |
| **临考期** | < 2 月 | 极度焦虑，求稳 | 保分策略 + 易错点复习 + 考场技巧 |
| **模考后** | N/A（事件触发） | 刚拿到成绩，最强动力 | "一模 XX 分能上什么学校" → 差距分析 → 精准补课 |

**当前产品已有**：prompt 里区分了三个时间阶段，这很好。
**缺失**：没有"模考后"这个最强转化场景的入口。

## 六、用户获取渠道分析

| 渠道 | 用户类型 | 转化路径 | 优先级 |
|------|----------|----------|--------|
| **家长群/朋友圈** | 焦虑型家长 | 截图分享 → 扫码/链接 → 试用 | ⭐⭐⭐⭐⭐ |
| **小红书/知乎** | 自驱型学生+家长 | "中考数学提分"搜索 → 文章 → 试用 | ⭐⭐⭐⭐ |
| **辅导老师推荐** | 被动型学生 | 老师发链接 → 学生使用 → 家长付费 | ⭐⭐⭐⭐ |
| **学校合作** | 全校学生 | 年级统一使用 → 家长续费 | ⭐⭐⭐ |
| **百度/抖音广告** | 混合 | 广告 → 落地页 → 注册 | ⭐⭐ |

**核心洞察**：教育产品最强的获客方式是**口碑传播**。需要设计"可分享的结果页"：
- 家长分享：一张带学校目标的诊断卡片
- 学生分享："本周函数从 L1 升到 L2" 的成就卡片

## 七、竞品对标用户体验

| 维度 | 本产品 | 猿辅导AI规划 | 作业帮学情报告 | 好未来智能诊断 |
|------|--------|-------------|---------------|---------------|
| 首次输入门槛 | 高（7模块自评） | 低（做3道题） | 无（自动采集） | 中（10题测试） |
| 结果可信度 | 中（依赖自评） | 高（题目测评） | 高（作业数据） | 高（自适应测试） |
| 个性化程度 | 高（区+校+模块） | 中 | 低 | 中 |
| 可执行性 | 高（具体到页码） | 中（推课为主） | 低（只诊断） | 中 |
| 持续使用 | ❌ 一次性 | ✅ 课程体系 | ✅ 持续跟踪 | ✅ 周测 |
| 知识库深度 | ⭐⭐⭐⭐⭐（5年真题+模考） | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ |

**本产品的差异化优势**：
1. 知识库深度——5年真题逐题分析 + 四区录取数据，这是独一无二的
2. 规划的可执行性——精确到课本页码和题号
3. 目标学校驱动——从"想上哪个学校"倒推数学需要多少分

**本产品的核心劣势**：
1. 输入靠自评，不是靠做题——可信度低
2. 一次性使用，没有留存机制
3. 没有家长专属入口

## 八、建议的用户画像优先级

### Phase 1（当前阶段，MVP 优化）

**主画像：焦虑型家长 + 中等层学生（60-80分）组合**

这是最大的用户群体，也是转化率最高的组合：
- 家长有动力（焦虑）
- 学生有空间（提分潜力大）
- 产品有数据支撑（目标校对标）

**具体建议**：

1. **加"快速入口"**：首页增加第四个卡片 —— "输入分数，看目标学校"
   - 只需 3 个输入：区、分数、想上什么档次的学校
   - 输出：可冲/稳妥/保底 三档学校 + 数学需要提多少分
   - 这是拉新的"钩子"

2. **简化全面规划表单**：
   - 7 模块自评改为可选项（默认"我不确定"→ 系统按总分推算）
   - 或者：先给一个基于总分的粗略规划，然后建议做 10 道快速测评题来精调

3. **加"分享结果"按钮**：
   - 诊断概览卡片 → 生成图片 → 一键保存/分享
   - 内容："XX区 · 目标八十中 · 当前72分 · AI诊断：函数和圆需重点突破 · 预计可提15分"

### Phase 2（有用户基础后）

**增加留存机制**：
- 学习打卡（每天标记完成的任务）
- 周报（本周完成率 + 建议下周调整）
- 模考后"成绩更新 → 重新诊断"

### Phase 3（规模化）

**多角色支持**：
- 家长端：看学习进度、接收周报
- 老师端：批量学生管理、导出报告
- 学生端：每日任务 + 打卡 + 积分

## 十、数据验证建议

在投入开发前，建议用以下方式验证用户画像假设：

1. **在 3-5 个家长群发产品链接**，观察：
   - 谁打开了？家长还是学生？
   - 在哪一步流失？（埋点：首页→选入口→填表→提交→看结果）
   - 看完结果后有没有人问"怎么让孩子用"？

2. **找 5 个真实初三家长做 15 分钟访谈**，验证：
   - "你平时怎么了解孩子的学习情况？"
   - "如果有个工具能告诉你孩子数学哪里差，你愿意用吗？"
   - "你觉得这个规划结果可信吗？为什么？"

3. **找 5 个初三学生做使用测试**，观察：
   - 7 模块自评时犹豫多久？改了几次？
   - 看到规划结果的第一反应是什么？
   - 会不会按计划执行？为什么不会？

## 附录：四个典型用户故事

### 用户 A：海淀区张妈妈
> 孩子初三，一模数学 68 分，目标育英学校（参考分 85）。晚上 10 点在手机上搜到本产品。
> **期望路径**：输入 68 分 + 海淀 → 看到"育英需要 85 分，差 17 分" → 看到最需要补的是函数和圆 → 分享给孩子 → 孩子按计划练
> **实际体验**：打开产品 → 看到 3 个入口 → 不确定选哪个 → 选了"全面规划" → 看到 7 个模块自评 → 不知道孩子函数到底"薄弱"还是"还行" → 随便选了 → 看到规划 → 不知道怎么发给孩子 → 截屏 → 关闭

### 用户 B：朝阳区小王同学
> 初三，最近月考数学 75 分，知道自己方程应用题不行。
> **期望路径**：打开产品 → 选"专项突破" → 选方程 → 开始练
> **实际体验**：✅ 基本匹配，drill 模式体验较好。但看完计划后没有后续跟踪。

### 用户 C：西城区李老师
> 带 8 个初三学生一对一辅导，想给家长一份专业的学习规划报告。
> **期望路径**：逐个输入学生信息 → 生成 8 份报告 → 导出 PDF → 发给家长
> **实际体验**：只能一个一个填，没法保存，没法导出，每次都要重新来。

### 用户 D：东城区小刘同学（被动型）
> 初三，数学 52 分，被妈妈要求打开这个产品。
> **期望路径**：妈妈已经帮他填好了信息，他只需要看每天做什么。
> **实际体验**：妈妈填不了自评（不了解每个模块），随便填的。计划里有"压轴题训练"——52 分的学生根本不需要做压轴题。
