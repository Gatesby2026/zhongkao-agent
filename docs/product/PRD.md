# 中考智能学习规划 — 产品需求文档（PRD）

> 版本：v5.0 | 更新日期：2026-05-10
> 产品定位：面向北京中考生家长的 AI 私人教研组
> 核心原则：**通过微信对话，持续陪伴学生备考全程**
> 交付形态：微信服务号「中考私人教研组」（流量入口/触达）+ 微信小程序（主对话载体，自建群聊 UI）

> **v5.0 重大架构变更**：从「个人微信群 + iLink Bot API」全面改为「服务号 + 小程序群聊 UI」。原因：iLink/ClawBot 设计为"个人账号连接 AI Agent"，不支持"多用户连一个 Bot"；企业微信外部群官方禁止机器人。详见 `~/.claude/projects/-Users-jiakui-projects-zhongkao-agent/memory/wechat-platform-decision.md`（仓库外的本地决策记录）。

## 一、产品定位

### 1.1 一句话描述

家长关注服务号、点开小程序，就拥有一个专属于孩子的"私人教研组"——班主任和各科老师在群聊式界面里持续跟进、主动推送、随时答疑。

### 1.2 与传统产品的本质区别

| 维度 | 传统学习 App | 本产品 |
|------|------------|--------|
| 入口 | 下载 App / 打开网页 | 微信扫码 → 服务号 → 小程序（零安装） |
| 交互 | 用户主动操作 | 老师主动触达（订阅消息推送） |
| 建档 | 填写问卷表单 | 对话中自然收集 |
| 感受 | 使用工具 | 有教研组在跟着 |
| 留存 | 靠用户习惯 | 靠老师推送触达 + 群聊氛围 |

### 1.3 家长核心诉求（五问）

产品所有对话设计都围绕回答这五个问题：

| # | 家长的问题 | 产品如何回答 |
|---|-----------|------------|
| Q1 | "我孩子 395 分，能上八十中吗？差多少？" | 班主任收集成绩后直接给出学校匹配和差距 |
| Q2 | "差的这些分，从哪里补最有效？" | 班主任给出各科提分性价比分析和方案 |
| Q3 | "具体每天做什么？" | 班主任每日推送学习任务 |
| Q4 | "这周做了没？进步了没？" | 班主任每周推送学习报告 |
| Q5 | "这个方案靠谱吗？" | 各科老师专业解答，数据支撑 |

## 二、考试基础

### 2.1 2026 北京中考方案

**总分 510 分，6 门计分科目**（2025 年改革后方案）：

| 科目 | 满分 | 构成 | 备注 |
|------|------|------|------|
| 语文 | 100 | 笔试 100 | 闭卷，150分钟 |
| 数学 | 100 | 笔试 100 | 闭卷，120分钟 |
| 英语 | 100 | 笔试 60 + 听说机考 40 | 听说两次机会取高分 |
| 物理 | 80 | 笔试 70 + 实验操作 10 | 闭卷，70分钟 |
| 道德与法治 | 80 | 笔试 70（开卷）+ 综合素质评价 10 | 开卷，70分钟 |
| 体育与健康 | 50 | 现场考试 30 + 过程性考核 20 | |

- **考查科目**（不计入总分，仅评等级 A/B/C/D）：历史、地理、化学、生物
- 文化课考试时间：6 月 24-25 日

### 2.2 各科提分策略

| 科目 | 提分特点 | AI老师策略 |
|------|---------|-----------|
| 数学 | 模块化强，差距大，提分空间大 | 题库诊断 + 精确薄弱点定位 + 每日推题 |
| 英语 | 听说 40 分模式固定可突破 | 听说模拟 + 阅读专项 |
| 物理 | 公式+实验，有规律可循 | 公式体系 + 实验操作要点 |
| 道法 | 开卷，答题框架 > 死记硬背 | 框架整理 + 材料分析方法 |
| 语文 | 高分段提分难，作文主观 | 维持为主，作文素材积累 |
| 体育 | 体能训练，非知识类 | 不配专属老师，班主任简单建议 |
| 考查科目 | 过等级即可，非重点 | 班主任兼任，策略性应对 |

## 三、用户与场景

### 3.1 目标用户

| 用户 | 画像 | 核心诉求 |
|------|------|---------|
| **主用户：家长** | 初三学生父母，焦虑但不知如何帮 | "孩子能上什么学校？怎么补？" |
| **执行用户：学生** | 初三学生，自驱或被安排 | "今天具体做什么？" |

> **设计优先级：家长 > 学生。** 家长是决策者、付费者、传播者。

### 3.2 使用场景

- 家长：晚上刷手机时收到班主任的学习报告，随手回复
- 学生：早上在群里看到今日任务，放学后发题目照片问老师
- 老师（Bot）：主动在关键时间节点推送，被动随时回答问题

## 四、产品形态

### 4.1 整体架构：服务号 + 小程序双层

```
┌──────────────────────────────────────────┐
│ 入口/触达层：微信服务号「中考私人教研组」      │
│  - 关注获取服务（家长扫码或公众号搜索）         │
│  - 长期订阅消息推送（教育类目，无 48h 限制）    │
│  - 智能回复处理简单问答                       │
│  - 菜单一键进入小程序                         │
└──────────────────────────────────────────┘
                  ↓
┌──────────────────────────────────────────┐
│ 主对话层：微信小程序「[学生]的私人教研组」      │
│  - 自建群聊 UI（仿微信群聊体验）              │
│  - 6 个 AI 老师独立头像、独立昵称同屏发言      │
│  - 富媒体：错题本、学习报告、解题动画           │
│  - 学情数据可视化页面                         │
└──────────────────────────────────────────┘
                  ↓
后端：多 Agent 协作（班主任 Orchestrator + 5 学科 Agent）
```

家长打开小程序后，看到的是一个**仿微信群聊的对话界面**，但里面是多个 AI 老师各自独立的发言：

```
┌──────────────────────────────────────┐
│  ←  王子涵的私人教研组       👥6        │  ← 标题完全个性化

│ ☀️ 早！今天的学习任务：                  │

│ 🔤 英语·听说模拟  30 分钟              │
│ 做完把数学题发给王老师看看 💪           │
│                                       │
│                          [家长]        │
│                  孩子今天状态不太好     │
│                  能不能少一点任务？     │
│                                       │
│ [李老师·班主任]                        │
│ 没问题，今天减半，保持状态比数量更重要   │
│ 明天补回来就好 💛                       │
│                                       │
│                          [学生]        │
│                  [发送一张数学题照片]   │
│                                       │
│ [李老师·班主任]                        │
│ 这是数学题，让王老师来看看 →            │
│                                       │
│ [王老师·数学]                          │
│ 这道题是配方法的应用，先把表达式...     │

└──────────────────────────────────────┘
[输入框]                          📷 ⓘ
```

**关键体验设计**：
- 每个老师独立头像、姓名 + 学科签名
- 老师之间可以"互动"（班主任 → 转给学科老师）
- 系统消息（任务推送、报告通知）独立样式
- 富媒体卡片可点击跳转详情页

### 4.3 服务号在整个产品中的角色

服务号**不承载主要对话**，定位是"**通知中心 + 流量入口**"：

| 服务号场景 | 触发方式 | 内容示例 |
|----------|---------|---------|
| 关注后欢迎语 | 家长关注 | 引导填写学生姓名 → 跳转小程序 |
| 长期订阅推送 | 家长授权后定期 | 每日学习计划、错题练习提醒 |
| 模板消息（重要通知） | 关键节点 | 一模/二模成绩录入提醒、考前倒计时 |
| 智能回复（兜底） | 家长在服务号窗口直接说话 | "去小程序里跟班主任聊详细的"按钮 |
| 菜单 | 始终可见 | "进入教研组" / "学情报告" / "联系真人客服" |

### 4.4 MVP 形态：单 Agent 群聊 UI（班主任 + 文字标签模拟多老师）

MVP 阶段先做**最小可验证版本**：

- 完整服务号 + 小程序框架
- 小程序内的群聊 UI 上线
- 但后端**只有班主任 Agent**，学科老师"出场"通过班主任口吻里的 `[转给王老师 → ]` 文字格式模拟
- 验证家长付费意愿和留存后，再拆分成真正的多 Agent 协作

### 4.5 两阶段路线图

| 阶段 | 形态 | 目标 |
|------|------|------|
| **MVP** | 服务号 + 小程序，群聊 UI 单 Agent | 验证家长是否愿意为"教研组陪伴"付费 + 留存 |
| **正式版** | 多 Agent 协作，真正多老师独立发言 | 完整教研组体验，多学科专精深度 |

## 五、角色设计

### 5.1 班主任（Orchestrator）

班主任是整个系统的核心，承担以下职责：

**路由职责**：
- 所有消息默认由班主任接收
- 用户未 @特定老师时，班主任判断并指派给对应科目老师
- 考查科目（历史/化学/生物/地理）由班主任直接回答

**管理职责**：
- 维护和更新学生档案
- 主动发起对话收集信息（建档期间）
- 监控学习进度，发现异常主动关心

**推送职责**：
- 每日学习任务推送
- 每周学习报告
- 阶段里程碑提醒（一模前、二模前、冲刺期）
- 基于错题数据的举一反三推送

### 5.2 科目老师

| 老师 | 科目 | 专长 |
|------|------|------|
| 语文老师 | 语文 | 阅读理解、作文、古诗文 |
| 数学老师 | 数学 | 七大模块诊断、题目讲解、举一反三 |
| 英语老师 | 英语 | 听说机考、阅读、完形 |
| 物理老师 | 物理 | 公式推导、实验操作、计算题 |
| 道法老师 | 道德与法治 | 开卷答题框架、材料分析 |

**科目老师响应规则**：
1. 班主任 @科目老师 → 科目老师接管并回复
2. 用户直接 @科目老师 → 科目老师直接回复
3. 科目老师回复后，班主任可补充总结或跟进

### 5.3 老师"账号"规划（小程序前端实现）

老师不再对应真实的微信账号，**全部由小程序前端 + 后端 Agent 实现**：

| 角色 | 实现方式 | 说明 |
|------|---------|------|
| 班主任 | 后端 Orchestrator Agent + 前端固定头像/昵称 | 所有学生共用同一个 Agent，按 unionid 隔离上下文 |
| 语文/数学/英语/物理/道法老师 | 后端 5 个学科 Agent + 各自前端头像/昵称 | 同上，每个 Agent 服务所有学生 |
| **合计** | **6 个 Agent + 6 套头像/人设** | unionid 隔离学生上下文，互不干扰 |

每个老师的"人设"配置（头像、昵称、签名、说话风格）存储在数据库，前端渲染时按消息发送方加载。

## 六、对话流程设计

### 6.1 Onboarding：从扫码到开始服务

**Step 1 — 关注服务号**

家长扫小程序码 / 服务号码 / 朋友推荐 → 关注「中考私人教研组」服务号。

**Step 2 — 服务号欢迎语 → 跳转小程序**

```
[服务号·中考私人教研组]
👋 欢迎来到中考私人教研组！

我们为每个孩子组建一个由班主任 + 5 学科老师
组成的专属教研组，提供个性化中考辅导。

👇 点击下方按钮进入小程序，开始为孩子建立档案
[进入教研组 →]（跳转小程序）
```

**Step 3 — 小程序内填写学生姓名 + 班主任激活**

家长在小程序内填写学生姓名（如"王子涵"），系统创建专属教研组，**班主任在群聊 UI 里发出第一条消息**：

```
[李老师·班主任]
王子涵家长您好！我是班主任李老师，
很高兴认识您。从今天开始，我和各科老师
会在这里陪伴王子涵备考中考 🎯

先来认识一下——王子涵现在初几了？
```

**Step 4 — 授权订阅消息（关键步骤）**

首次对话后，弹出订阅消息授权弹窗：

```
[订阅消息授权]
中考私人教研组想要给您发送：
☑️ 每日学习任务提醒（长期）
☑️ 周学习报告（长期）  
☑️ 重要节点提醒（一模/二模/中考前）

[同意]  [取消]
```

家长授权后，后续可在不打开小程序时也收到推送。

**Step 5 — 持续建档对话**

家长在小程序的群聊 UI 中跟班主任继续对话，逐步完善学生档案。

### 6.2 建档对话：逐步了解学生

班主任通过自然对话收集信息，不做问卷填表。

**建档第一阶段（前3天，了解基本情况）**：
```
班主任: 王子涵现在初几了？

家长: 初三了

班主任: 好的！距离中考还有不少时间。
       上次模考（或期末考）总分大概多少？

家长: 395分左右

班主任: 395分，有目标学校吗？比如想冲哪所高中？

家长: 想上八十中

班主任: 八十中近几年录取线在450分左右，
       差距55分，有一定挑战但完全可以冲！
       各科成绩大概是多少？我帮你看看从哪里
       补最有效率……
```

**建档第二阶段（持续进行，对话中积累）**：
- 家长发试卷照片 → 记录错题和薄弱点
- 家长提到孩子某科弱 → 更新档案优先级
- 模考成绩出来 → 更新档案，重新评估方案

### 6.3 日常：主动推送

**每日早晨任务推送**（班主任）：
```
[班主任李老师]: ☀️ 早上好！今天的学习任务：

📐 数学 · 函数基础    45分钟
   目标：一次函数图象和性质
   
🔤 英语 · 听说模拟    30分钟

今天总计75分钟，加油 💪
做完可以把数学题发给王老师看看！
```

**举一反三推送**（数学老师，事件触发）：
```
[数学王老师]: 子涵昨天这道函数题做错了，
给你出一道类似的练习一下：

[题目图片]

思路提示：先找两点，再确定斜率……
```

**每周学习报告**（班主任，每周日晚）：
```
[班主任李老师]: 📊 本周学习报告（5/4-5/10）

完成情况：计划14项，完成11项（79%）✅
数学正确率：68%，比上周提升9% 📈
学习时长：9.5小时

重点进展：
· 函数模块明显进步，基本概念清晰了
· 英语听说还需加强，建议下周多练

距中考还有45天，子涵这周很努力！
继续保持 💪
```

### 6.4 日常：被动答疑

**学生发题目照片**：
```
[学生]: [发送题目照片]

[班主任李老师]: 这是数学题，@数学王老师 来看看

[数学王老师]: 好，我看一下……
这道题是圆的切线问题，解题步骤：
1. 先连接切点和圆心
2. 切线与半径垂直，所以角度关系……
```

**家长问学校信息**：
```
[家长]: 八十中今年录取分数线多少？

[班主任李老师]: 根据近几年数据：
· 2025年：452分（海淀区）
· 2024年：448分
· 2023年：455分
子涵现在395分，差距约55分，
建议重点提数学和英语，这两科空间最大。
```

## 七、学生档案

### 7.1 档案结构

通过对话积累的核心数据：

```
学生档案（student_profile）
├── 基本信息
│   ├── 姓名、年级
│   ├── 目标学校
│   └── 所在区
├── 成绩数据
│   ├── 各科当前分数
│   ├── 总分
│   └── 历次模考记录
├── 学科画像
│   ├── 数学：7模块掌握程度
│   ├── 英语：听说/阅读/写作评估
│   └── 其他各科：薄弱点标签
├── 错题库
│   └── 按科目/知识点分类
├── 执行记录
│   ├── 每日任务完成情况
│   ├── 学习时长
│   └── 连续打卡天数
└── 对话摘要
    └── 班主任提炼的关键信息
```

### 7.2 档案完整度

档案越完整，推送越精准：

| 完整度 | 状态 | 班主任行为 |
|--------|------|-----------|
| 刚入群 | 只知道姓名 | 主动提问收集基本信息 |
| 了解基本成绩 | 可给粗略方向 | 开始推送通用任务 |
| 完成科目诊断 | 可给精确计划 | 推送个性化任务 |
| 有持续错题数据 | 高度个性化 | 精准举一反三 |

### 7.3 数据来源

| 数据 | 来源方式 |
|------|---------|
| 基本信息 | 班主任主动问 |
| 各科成绩 | 家长告知 / 模考后更新 |
| 数学薄弱点 | AI分析错题照片 |
| 错题记录 | 学生发题目照片 → Bot批改记录 |
| 执行数据 | 家长/学生在群内确认完成 |

## 八、技术架构

### 8.1 微信接入层

| 项目 | 方案 |
|------|------|
| 入口主体 | 微信服务号 + 微信小程序（同企业主体下） |
| 资质要求 | 营业执照（已具备）+ 服务号企业认证（300 元/年）+ 小程序注册 |
| 服务号能力 | 智能回复、模板消息、菜单跳转小程序 |
| 小程序能力 | 自定义 UI（含群聊 UI）、长期订阅消息、客服会话兜底 |
| 主动推送 | 小程序长期订阅消息（教育类目，无 48h 限制）+ 服务号模板消息 |
| 消息支持 | 文字、图片、语音、文件（小程序内全支持） |
| 补贴 | 申请 2026 AI 小程序成长计划：1 亿混元 token + 6 个月免费云开发 |

### 8.2 不再使用的方案（避免重复踩坑）

| 已废弃方案 | 废弃原因 |
|----------|---------|
| ❌ 个人微信 iLink Bot API | 设计为"个人账号 ↔ 自己的 AI Agent"，不支持多用户连一个 Bot |
| ❌ 企业微信外部群 + 群机器人 | 官方明令禁止机器人加入含微信用户的外部群 |
| ❌ 企业微信 + 会话存档 | 5400 元/年/班且只读不能发，必须真人值班 |
| ❌ 个人微信第三方 hook | 灰产，违反协议，封号风险 |

### 8.3 后端架构

```
入口层
├── 服务号事件回调（关注/订阅/客服消息/模板消息回执）
└── 小程序 WebSocket / 长连接（实时聊天）
        ↓
消息路由层（统一）
├── 用户身份识别（unionid 跨服务号/小程序识别同一家长）
├── 学生档案匹配（unionid → student_profile）
├── 意图识别 → 路由给对应 Agent
└── 兜底交给班主任 Agent
        ↓
Agent 层（多租户、单实例多会话）
├── 班主任 Agent（Orchestrator）
│   ├── 读取/更新学生档案
│   ├── 调用各科 Agent
│   ├── 管理推送调度
│   └── 在小程序群聊 UI 中以"班主任"身份发言
└── 科目 Agent × 5（语数英物道）
    ├── 学科专业知识库
    ├── 错题分析能力
    └── 在小程序群聊 UI 中各自独立头像/昵称发言
        ↓
推送调度层
├── 定时任务（每日任务、周报、考前提醒）
├── 事件触发（新错题→举一反三、状态异常→班主任关心）
└── 通过订阅消息/模板消息触达
        ↓
存储层
├── 学生档案 DB（按 unionid 多租户隔离）
├── 错题库
├── 群聊消息历史（按学生 unionid 隔离）
└── 知识库（现有 zhongkao-agent 内容）
```

### 8.4 多租户隔离

- 后端是**单实例多会话**架构（同一套 Agent 服务所有学生）
- 通过家长 **unionid**（微信开放平台跨应用统一 ID）匹配学生档案
- 同一个"王老师 Agent"为所有学生服务，每次推理时载入对应学生上下文
- 各学生数据完全隔离，前端群聊消息历史按 unionid 分组

### 8.5 技术栈

| 层 | 技术 |
|----|------|
| **小程序前端** | 微信小程序原生开发（或 Taro）+ 自建群聊 UI 组件 |
| **服务号接入** | 微信公众平台 SDK |
| **后端语言** | Node.js（与小程序云开发兼容好）或 Python |
| **后端基础设施** | 微信云开发（MVP，免运维 + 6 个月免费）→ 自建（规模化后） |
| **AI 模型** | Claude / 混元（云开发免费 token）/ MiniMax / DeepSeek |
| **视觉模型** | Qwen-VL-Max（已验证，用于答题卡 OCR） |
| **数据库** | 云开发 NoSQL（MVP）→ PostgreSQL（正式版） |
| **推送调度** | 云开发定时触发器 + 事件队列 |
| **知识库** | 现有 zhongkao-agent 数学题库 + 录取分数线（已结构化） |

> **Web 端**：现有 Next.js 产品保留，定位为**内部管理后台**（查看学生档案、数据统计、运营管理、内容编辑），不作为家长用户界面。

## 十、设计原则

1. **对话即产品**：不依赖用户主动打开 App，通过推送触达用户
2. **老师人格化**：每个 Bot 有名字、有风格，不是冷冰冰的机器人
3. **主动 > 被动**：班主任主动推送 > 等用户来问
4. **积累 > 一次性**：每次对话都在丰富档案，推送越来越精准
5. **结果承诺**：每个方案都要有量化的预期（"4周后数学预计提升8-12分"）
6. **通俗表达**：不出现技术标签，用家长听得懂的语言
7. **移动优先**：微信就是移动优先
8. **数学先行**：数学有完整题库和AI能力，优先做深
9. **紧迫感**：距中考天数始终是对话背景
10. **信任积累**：宁可少说，不说没依据的话
