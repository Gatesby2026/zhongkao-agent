## 一、原始问题与之前的失败路径

| 路径 | 困难 |
|------|------|
| 商业 OMR API | 国内三大云只切印刷题，不识别涂卡 |
| LLM 直接标 bubble 坐标 | LLM-vision 不擅长精确像素定位，幻觉 |
| OpenCV 自动找黑色方块 | 100+ 假阳性，需要 region 约束 |
| OMRChecker per-card template | 100+ 套卡需手工配置 |
| Set-of-Marks hybrid | 复杂工程，但 LLM 仍要做精确定位 |

**所有这些方法都把"找出涂了哪格"当成图形理解问题**——这恰好是 LLM-vision 的弱项。

---

## 二、范式转换

### 重新看答题卡的物理本质

每道涂卡题印刷模板：

```
2. [ A ] [ B ] [ C ] [ D ]
```

学生涂卡的物理行为：**用 2B 铅笔把字母覆盖**——盖住字母 = 完成填涂。

考试规则甚至明确写："**黑度以盖住框内字母为准**"。

### 由此推出

涂黑的格子里 = OCR 读不到字母。
**翻转任务**：
- 不去看"哪个 bubble 被涂了"（图形识别 - LLM 弱）
- 而是看"哪个字母 OCR 没读到"（字符识别 - LLM 强）

OCR 输出 `"2. B C D"` → A 没了 → 学生涂了 A。

## 四、为什么这次 LLM 能给对答案？

之前 LLM 失败的所有任务（标 bubble、标 ABCD 字母坐标）都是**"在二维空间里精确定位抽象对象"**。

这次的任务变成了**"读出印刷文字"**——
- 字符识别是 LLM-vision 的核心训练目标
- 而且只需要"读得到/读不到"的二值判断，**根本不需要坐标**

任务复杂度从 O(图形理解) → O(文字识别)，难度降一个数量级。

## 五、覆盖范围

| 情形 | 这个方法能覆盖？ |
|---|---|
| 标准 ABCD 单选 | ✅ |
| 多选题（如 ABD） | ✅ 缺 2-3 个字母都能推 |
| ABCDE 5 选项 | ✅ 改 regex |
| 多题在一行（如 "13. A C D 14. A C D"） | ✅ 实测正常 |
| 嵌套子题（"15.① A C D ② A B D"） | ⚠️ regex 要细化 |
| 字符填涂题（Q1 那种"赤城书乡"汉字格） | ✅ 不匹配 ABCD 模式，自动跳过 |
| 学生没涂任何选项 | ✅ OCR 读到 4 个字母 → 推断"未作答" |
| 学生涂得不够黑、OCR 仍读到字母 | ⚠️ 误判为"未作答"——会被兜底检测 |
| 学生涂错地方（涂到 bubble 外） | ⚠️ 类似上面 |
| 答题卡折角 / 油渍遮挡部分 | ⚠️ OCR 整体降级 |

**核心覆盖率：标准印刷 ABCD 涂卡 100%。** 边缘 case 需要兜底逻辑。

## 八、产品路线全面修正

### 之前（路线 B）
- 8-12 周 MVP
- 维护 100+ 套 template
- 兜底 UI 让家长确认 10% 题
- 准确率 ~90%

### 现在（修正版）
- **1-2 周 MVP**：缺字母法 + 简单边缘 case 处理
- **零 template** 维护
- **兜底 UI 只为极端 case**（折角 / 涂得太浅）：< 5% 题
- **准确率预期 95%+**

### 完整 pipeline

```
家长拍答题卡
    ↓
[预处理] 模糊检测、亮度归一化（OpenCV）
    ↓
[OCR] qwen-vl-ocr-latest 全图 OCR
    ↓
[Regex] 抽取 "数字. ABCD字母组合" 模式
    ↓
[推断] 每题缺哪个字母 = 涂了哪个
    ↓
[置信度]
    缺 1 个 → 单选答案，高置信
    缺 2-3 个 → 多选答案，高置信
    缺 0 或 全部缺 → 异常，进兜底 UI
    ↓
[输出] answer-card.json（直接喂给已有 student-report pipeline）
```
