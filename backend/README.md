# 后端「中考私人教研组」

> 状态：📋 规划中，等待 PRD 设计细化后启动开发

## 这是什么

服务于「中考私人教研组」小程序 + 服务号的统一后端。

详见 [`../docs/product/PRD.md`](../docs/product/PRD.md) v5.0「八、技术架构」章节。

## 职责

- 接收服务号事件回调（关注/订阅/客服消息）
- 接收小程序 API 调用（聊天消息、加载历史、订阅授权回调等）
- 多 Agent 路由与执行（班主任 + 5 学科老师）
- 学生档案管理（多租户，按 unionid 隔离）
- 推送调度（每日任务、周报、举一反三、节点提醒）
- 错题库管理

## 技术栈

- **运行时**：MVP 阶段在微信云开发（Node.js + 云函数）→ 规模化后迁自建
- **语言**：Node.js（与云开发兼容好）
- **AI**：Claude / 混元（云开发免费 token）/ MiniMax / DeepSeek
- **视觉**：Qwen-VL-Max（已有 OCR 能力）
- **DB**：云开发 NoSQL（MVP）→ PostgreSQL（正式版）
- **知识库**：复用 `../knowledge-base/` 已结构化的中考资料

## 与现有 `admin/` 目录的关系

- `admin/`：现有 Next.js Web 端，定位为**内部管理后台**（不是家长用户界面）
- `backend/`：本目录，是**家长用户的后端服务**（响应小程序与服务号请求）

两者数据共享但职责不同。
