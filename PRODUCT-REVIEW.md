# PRODUCT-DESIGN.md 评审报告

> 评审人：资深教育行业产品经理
> 评审对象：PRODUCT-DESIGN.md v2.1
> 评审日期：2026-04-12
> 评审视角：家长使用场景优先，兼顾学生和商业可行性

---

## 总体评价：65 分 / 100

文档在**技术架构和数据闭环**设计上比较完整，能看出工程师主导的思维。但作为一份产品设计文档，存在几个根本性问题：

**核心问题：这份文档描述的是"系统能做什么"，而不是"用户要解决什么问题"。**

---

## 一、致命缺陷（必须修正）

### 1.1 缺少「家长的核心场景」定义

文档通篇在讲三层能力（查漏补缺 / 单科规划 / 全科规划），但没有回答一个基本问题：

> **家长打开这个产品，到底要解决什么问题？**

家长不会想"我要做一个查漏补缺"。家长的真实想法是：

| 家长真实问题 | 产品当前对应 | 差距 |
|-------------|-------------|------|
| "我孩子 78 分，能上八十中吗？差多少？" | /score-check ✅ | 基本满足 |
| "差的这些分，从哪里补最有效？要多久？" | /drill + /plan ⚠️ | **缺少直接回答** |
| "具体每天做什么？怎么执行？" | 🔲 未实现 | **最大断层** |
| "这周做了没？做得怎么样？进步了没？" | 🔲 未实现 | **留存关键** |
| "离中考还有 73 天，来不来得及？" | 倒计时 ⚠️ | 有倒计时但没有关联到"来得及"的信心建立 |
| "这个计划靠谱吗？凭什么信你？" | 🔲 未回答 | **信任建立为零** |

**建议**：文档第一章应该从「家长五问」出发，然后说明产品如何一步步回答每个问题。三层能力模型是实现手段，不是产品定位。

### 1.2 score-check 的分数体系错位

当前 score-check 页面输入的是**数学单科分数（0-100）**，匹配学校用的也是单科百分制。但家长关心的是：

> "我孩子总分 395，能上什么学校？"

**问题**：
- 家长不会用数学单科分去匹配学校，他们想的是总分
- 中考录取线是按总分 510 划的，不是按单科
- 当前页面没有说清楚输入的是"数学分"还是"总分"

**建议**：score-check 应支持两种模式：
- 快速模式：输入总分（510 制），匹配学校
- 详细模式：输入各科成绩，自动算总分，再匹配学校

### 1.3 「了解学生」和「给出计划」之间缺少关键一步 —— 可信的解读

当前流程是：测评 → 得到 L0/L1/L2 → 生成计划。

**问题**：家长看到"函数 L0"时的反应不是"好，我知道了"，而是：

- "L0 是什么意思？严重吗？"
- "其他孩子都什么水平？我孩子算差的吗？"
- "这个 L0 提上去大概能涨多少分？"
- "为什么你说是 L0？依据是什么？"

当前产品**没有解读层**。诊断结果直接跳到计划，跳过了**让家长理解和建立信任**的过程。

**建议**：在诊断和计划之间增加「诊断解读页」：
- 每个模块用通俗语言说明情况（"函数部分基础概念不清楚，看到 y=kx+b 不知道 k 和 b 代表什么"）
- 标注该模块在中考中占多少分（"这部分通常考 15-20 分"）
- 给出提升预期（"补上这块预计可以多拿 8-12 分"）
- 给出信心（"按我们的方案，80% 的类似水平学生在 3 周内提升了一个等级"）

## 二、重大缺陷（强烈建议修正）

### 2.1 三层能力之间缺少引导逻辑

文档把三层能力平铺在首页让用户选，但**家长不知道该选哪个**。

- "查漏补缺"和"数学完整规划"有什么区别？
- 我应该先做哪个？
- 做完"查漏补缺"还需要做"数学完整规划"吗？

**当前设计像一个工具箱，而不是一个顾问。**

**建议**：不要让用户选择路径，而是由系统根据用户状态自动推荐：

```
无数据的新用户 → 引导做测评 → 测评完自动进入"数学完整规划"
                              → 规划中自动包含查漏补缺的内容

有画像的老用户 → 首页直接显示"本周重点"
              → 提供"调整计划"入口而不是"重新做规划"

考前 30 天    → 自动推荐全科冲刺方案
```

查漏补缺不应该是一个独立入口，而应该是"数学规划"的子能力。家长不需要理解你的产品架构。

### 2.2 缺少「结果承诺」

整个文档没有一处告诉用户：**用了这个产品，预期会怎样。**

家长需要的是：
- "按这个方案执行 4 周，数学预计提升 8-15 分"
- "你孩子目前 395 分，按方案执行到中考，总分预期达到 430-450 分"
- "达到八十中录取线的概率：60%（当前） → 85%（按方案执行后）"

**没有结果承诺的计划，家长不会执行。** 他们见过太多空洞的学习计划了。

**建议**：每个计划都要有量化的预期产出，并标注置信度和条件（"基于每天执行 45 分钟的前提"）。

### 2.3 缺少「为什么信任你」

文档底部写了"数据覆盖：海淀 / 西城 / 东城 / 朝阳 四区 | 2021-2025 五年真题 140 道"。

这远远不够。家长的信任建立需要：

| 信任要素 | 当前状态 | 建议 |
|---------|---------|------|
| 数据来源可信 | 仅一行文字 | 专门的「数据说明」页面，列出试卷来源和数量 |
| 诊断准确 | 无验证 | 展示"诊断结果 vs 真实模考成绩"的吻合度 |
| 计划有效 | 无案例 | 展示用户案例（可匿名）："使用 4 周后数学提升 12 分" |
| 专业背书 | 无 | "方案基于海淀区一线教师的教学经验" |
| 产品透明 | 部分 | 解释"为什么推荐先补函数"而不是直接给结论 |

### 2.4 家长端和学生端没有区分

家长和学生的需求完全不同：

| 维度 | 家长 | 学生 |
|------|------|------|
| 核心需求 | "孩子什么水平？能上什么学校？怎么帮？" | "今天具体做什么题？" |
| 使用场景 | 晚上看手机研究，周末和孩子讨论 | 坐在书桌前执行 |
| 关注指标 | 总分变化、学校匹配、进步趋势 | 今日任务、正确率、错题 |
| 操作频率 | 每周 1-2 次 | 每天 |
| 信息密度 | 要概览和结论 | 要具体和可操作 |

**当前产品把两者混在一起**。家长不需要看"L0/L1"的技术标签，学生不需要看学校匹配。

**建议**：不必做两套完全独立的界面，但至少要：
- 注册时选择角色（家长/学生）
- 首页根据角色显示不同重点
- 家长视角：概览仪表盘（总分趋势+学校差距+本周完成率）
- 学生视角：今日任务+刷题入口

## 三、中等问题（建议修正）

### 3.1 首页信息过载

当前首页设计有 3 个分区、8+ 个入口卡片。对于一个初次使用的焦虑家长来说，**选择太多 = 选择困难 = 离开**。

**建议**：首页只做两件事：
- 新用户：一个入口（"3 分钟了解孩子的真实水平" → 测评）
- 老用户：一个入口（"继续本周计划" / "查看最新进度"）

其他功能放到二级页面。

### 3.2 测评只有数学

10 题测评目前只覆盖数学。但家长的焦虑不止数学。

**短期建议**：测评结果页明确说明"本次测评仅覆盖数学，其他科目规划基于你输入的成绩估算"，管理预期。

**中期建议**：增加各科简单自评（比模块自评更粗，只问"最近一次考试成绩"和"自我感觉"），至少让全科规划有基础数据。

### 3.3 "画像完整度"对家长无意义

文档很强调画像完整度（22%、55%...），但家长不关心这个数字。

家长关心的是："你是不是已经足够了解我的孩子，可以给出靠谱的建议？"

**建议**：把技术指标翻译成用户语言：
- 22% → "我们对你孩子的了解还不够，建议做一次测评"
- 55% → "基于测评数据，我们可以给出较准确的数学规划"
- 80% → "基于长期追踪数据，我们的规划精确度很高"

### 3.4 路径 C（考前冲刺）设计薄弱

距中考 73 天，这是当前最紧迫的场景。但全科规划还是"待实现"。

**现实问题**：现在是 4 月中旬，6 月 24 日中考。产品的目标用户**正在这个时间窗口**。如果全科规划 5 月才上线，错过了最需要的时间。

**建议**：优先级重排——全科规划的简化版（输入各科成绩 → 给出时间分配建议 + 数学详细计划）应该是 **P0**，比数学规划的"四步闭环升级"更紧急。

### 3.5 缺少模考成绩导入能力

北京初三学生最有价值的数据是**一模/二模成绩**。一模通常在 4 月底，二模在 5 月底。

当前产品完全没有对接这个场景。

**建议**：增加"导入模考成绩"功能：
- 输入各科分数 → 自动更新画像 → 重新生成规划
- 一模后是家长最焦虑的时间节点，也是最佳获客时机

## 四、优点（值得保持）

1. **数据闭环设计思路正确**：测评→画像→计划→执行→反馈 的飞轮是对的
2. **多来源置信度设计精巧**：source + confidence 的模型比简单的 level 好很多
3. **被动采集优于主动填表**：这个原则完全正确
4. **考试基础信息准确**：510 分 6 科的结构是对的
5. **分阶段实施的路线合理**：先做好一个科目再扩展

---

## 五、优先级建议（重排）

基于"距中考 73 天"的现实，重新排优先级：

### P0 — 本周必须做（让产品对当前用户有用）

| 任务 | 理由 |
|------|------|
| score-check 支持总分匹配 | 家长第一诉求，当前单科匹配不对 |
| 全科时间分配（简化版） | 输入 5 科成绩 → 时间分配建议 + 数学详细计划 |
| 测评/计划结果增加通俗解读 | 不能让家长看着 L0/L1 一脸懵 |
| 计划增加"预期提分"承诺 | 没有结果预期，家长不执行 |

### P1 — 两周内做（留存关键）

| 任务 | 理由 |
|------|------|
| 数据回写（测评/刷题→画像） | 闭环基础 |
| 一模成绩导入 | 4 月底一模是关键节点 |
| 周报（家长视角） | 家长需要看到进展 |
| 首页简化 + 角色分流 | 减少选择困难 |

### P2 — 一个月内做

| 任务 | 理由 |
|------|------|
| 每日任务视图 | 学生执行必需 |
| 数学规划四步闭环 | 长期价值 |
| 信任建设（数据说明页+用户案例） | 提高转化率 |

### 延后或砍掉

| 任务 | 理由 |
|------|------|
| 底部 Tab 导航 | 页面不多，暂时不需要 |
| 错题本 | 非核心，可后做 |
| LLM 对话诊断 | 好功能但不紧急 |
| BKT 模型 | 过度设计，当前用户量不支撑 |

---

## 六、总结

这份文档**技术功底扎实，但产品感不足**。

最大的gap是：**文档在描述一个工具系统，但家长需要的是一个顾问。**

工具思维："这里有测评、规划、刷题三个功能，你来选。"
顾问思维："你孩子数学 78 分想上八十中？差 12 分。我看了他的情况，函数和圆最弱，补上这两块能涨 10-15 分。按这个方案走，4 周后再看。"

**产品的每一个页面都应该在回答家长的一个具体问题，而不是在展示一个系统的功能。**
