## 三、各层详细说明

### ① 地区政策层

**现状：✅ 完整可用**

| 文件 | 内容 | 数据源 |
|------|------|--------|
| policy.yaml | 2026 年中考 6 科计分制度（510 分）、考试时间、录取批次（提前/校额到校/统招）、未来趋势 | 北京教育考试院 |
| districts/*.yaml | 各区教研特色、使用教材版本（海淀/西城/东城/朝阳 = 人教版） | 教委公开信息 |

**待改进：**
- [ ] 补充丰台、大兴等区（当前只有 4 区）
- [ ] 增加石景山/通州等使用北京版教材的区
- [ ] 各区模考时间、教研侧重点可进一步细化

### ② 学科考纲层

**现状：⚠️ 有已知错误需修正**

| 文件 | 内容 | 问题 |
|------|------|------|
| curriculum.yaml | 2022 课标知识点树 | 较完整 |
| exam-spec.yaml | 考试说明 | 需检查是否与 510 分制对齐 |
| question-types.yaml | 题型分布 | ❌ 总分仍写 120，应为 100 |
| weight-analysis.yaml | 知识点权重 | 需与 exam-analysis 层交叉验证 |

**待改进：**
- [ ] **question-types.yaml 总分改为 100**（当前错误地写了 120）
- [ ] 用 exam-analysis 层的 5 年真题数据回填权重，使其更精确

### ③ 诊断标准层

### ④ 学习路径层

**现状：✅ 可用，已修正为人教版章节**

7 个模块文件，每个包含 L0→L1→L2→L3 的升级路径，推荐资源已对齐人教版教材。

**待改进：**
- [ ] 教师审核路径是否符合实际教学经验
- [ ] 与 admission 层联动：根据目标学校倒推"你需要达到 L 几"
- [ ] 补充时间规划维度（距中考 X 个月，各阶段侧重点）

### ⑤ 真题分析层

**现状：✅ 刚建成，数据完整**

5 年（2021-2025）× 28 题 = 140 道题的逐题分析 + 跨年汇总。核心价值：

- **题号稳定性**：Q9（二次根式）、Q10（因式分解）、Q17（实数运算）等连续 5 年不变
- **分数段策略**：60 分保底哪些题、80 分需加哪些题、90+ 需突破哪些题
- **命题趋势**：实际情境题比例从 30% 升至 40%、函数占 30-35 分

**待改进：**
- [ ] 补充 2024 年逐题分析（当前 2024.yaml 已建但相对简略）
- [ ] 增加每题的得分率/区分度数据（如有来源）
- [ ] 与 diagnostics 层联动：某模块 L1 的学生预计能拿到 exam-analysis 中哪些分

## 四、数据质量标记

每个文件的数据可靠性分三级：

| 标记 | 含义 | 典型场景 |
|------|------|---------|
| 🟢 官方数据 | 来自教育考试院/教委/课标 | policy.yaml、curriculum.yaml |
| 🟡 网络数据 | 来自家长帮/菁优网/知乎，交叉验证过 | admission/*.yaml、exam-analysis/*.yaml |
| 🔴 LLM 生成 | Claude/GPT 生成，未经教师审核 | diagnostics/*.yaml、learning-paths/*.yaml |

**当前知识库可靠性分布：**
- 🟢 官方数据：policy.yaml、curriculum.yaml、textbooks.yaml（约 15%）
- 🟡 网络数据：exam-analysis/*、admission/*、weight-analysis.yaml（约 45%）
- 🔴 LLM 生成：diagnostics/*、learning-paths/*（约 40%）

**改进方向：通过教师审核，将 🔴 逐步升级为 🟡/🟢。**

## 五、各层之间的联动关系

```
用户输入"我想上八十中"
        │
        ▼
  ⑥ admission/chaoyang.yaml
  → 八十中普通班需 482/510（2025参考）
  → math_target: 90+
        │
        ▼
  ⑤ exam-analysis/summary.yaml
  → 90分策略：选择1-8 + 填空9-16 + 解答17-25 + 26/27前两问
  → 必须拿下的模块：函数(30-35分)、几何证明(25-30分)
        │
        ▼
  ② subjects/math/beijing/weight-analysis.yaml
  → 各模块分值权重 → 优先攻哪个模块
        │
        ▼
用户输入自评成绩
        │
        ▼
  ③ diagnostics/math/*.yaml
  → 各模块定级（如：函数 L1、几何 L2、统计 L2）
  → 诊断结论：函数是主要短板
        │
        ▼
  ④ learning-paths/math/beijing/functions.yaml
  → 函数 L1→L2 升级路径：
     步骤1：回归人教版八下第十九章
     步骤2：五三基础篇一次函数专题
     步骤3：真题 Q22（一次函数待定系数法）限时训练
        │
        ▼
  LLM 生成个性化报告
  → 结合以上所有数据，输出自然语言学习计划
```

## 八、数据采集方法论（供后续扩展参考）

### 各层的最佳数据源

| 层级 | 一手来源 | 二手来源 | LLM 辅助 |
|------|---------|---------|----------|
| ① 政策 | 教育考试院官网 | 中考网/公众号 | 提取结构化 |
| ② 考纲 | 课标 PDF + 考试说明 | 菁优网知识点树 | 知识点提取 |
| ③ 诊断 | 教师访谈 | 真题难度分层 | 生成初稿 |
| ④ 路径 | 教师设计 | 教辅目录体系 | 扩展细节 |
| ⑤ 真题 | 考试院真题 + 评析 | 菁优网/知乎解析 | 逐题标注 |
| ⑥ 分数线 | 考试院录取数据 | 家长帮/北京中考在线 | 整理结构化 |

### 核心工具

| 工具 | 用途 | 费用 |
|------|------|------|
| 菁优网 VIP | 知识点体系 + 真题标注 + 难度标记 | ~200 元/年 |
| 学科网 | 模拟题采集、试卷下载 | ~298 元/年 |
| 国家智慧教育平台 | 免费教学视频资源 | 免费 |

### 年度更新节奏

| 时间 | 动作 |
|------|------|
| 每年 3 月 | 更新 ① 政策层（新年度中考方案发布） |
| 每年 7 月 | 更新 ⑤ 真题层（当年中考结束后） |
| 每年 8 月 | 更新 ⑥ 分数线层（录取结果公布后） |
| 持续 | ③④ 根据用户反馈和教师审核迭代 |

## 九、资源推荐规则

```yaml
safe_to_recommend:        # 放心推荐
  - 对应版本教材（需匹配考生所在区的版本）
  - 五年中考三年模拟（五三）
  - 万唯中考系列
  - 天利38套
  - 国家中小学智慧教育平台视频
  - 各区一模二模真题
  - 近5年北京中考真题

need_verification:        # 确认后推荐
  - 各类网课平台（质量参差）
  - 小众参考书
  - 自媒体教学内容

never_recommend:          # 不推荐
  - 盗版资源
  - 未经验证的付费课程

matching_rules:           # 水平匹配
  - L0/L1 → 基础题为主，跳过难题
  - L2    → 中档题为主，适量难题
  - L3    → 真题+模拟题限时训练
  - 教材版本必须匹配考生所在区
```
