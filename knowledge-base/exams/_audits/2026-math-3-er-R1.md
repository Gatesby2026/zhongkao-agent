# 2026 北京数学二模 docx 路线 R1 审核（chaoyang / fengtai / yanshan）

**审核方式**：严格核源 docx（python-docx 读 paragraphs，零 OCR），并比对 yaml `idx → real Q#`。

## OVERALL: FAIL（3/3 都有 parser bug，必须 R2 修 parser 重跑，不可走 patches）

3 区源 docx 真实题数均为 **28 题**（与中考标准一致），yaml 多出来的 1 题全部是 parser bug，**没有任何区是真的多 1 题**。但 3 区的 bug 根因 **完全不同**，是 3 个独立 parser 缺陷在不同区的源 docx 上各自暴露。

## 1. chaoyang（29 题/106 分 = 28+1）

### Q29 真相
- 源 docx Q1-Q28 完整存在（L5=Q1 … L95=Q28）。
- yaml `idx24` stem = `"3  23.4  24.1  25.2  25.6  26.3  26.4  27.3  27.8 c．女员工数据是："` —— 这是 **Q23（统计应用题）正文里的 "b．男员工数据" 那一行数据被切成了独立题**。
- 之后 yaml `idx25..idx29` 才是真实的 Q24..Q28，整体右移 1 位。
- 所以 chaoyang yaml 实际是 `Q1..Q23 → idx1..idx23, [脏块] → idx24, Q24..Q28 → idx25..idx29`，**Q29 stem 内容是合理的（真 Q28 关联图形题），但题号位置错位**。

### Parser bug 定位
- Q23 段内出现 `b．男员工数据在这一组的是：` + 单独一行数字（L60-L61）。Parser 把 `b．` / 行首数据 `3  23.4…` 误识别成新题头（疑似 `\d+．` 容忍了 `b．` 或 OCR 风格的全角 b→数字混淆）。
- 实际不是 OCR — docx 源里 L60 就是 `b．`，**character `b` ≠ `\d`，应该不会触发数字题头 regex**。更可能是后续 normalize 把 `b．` 之后 L61 的 `3 23.4…` 当成 `3．` 题头（因为 `3` 后两空格再数字）。需 inspect 题头 regex 是否要求紧跟非空白且非数字。

### 跨区共性
仅 chaoyang 出现（其它 2 区 Q23 数据形态不同）。属 **chaoyang 单区源 docx artifact**，但建议 parser 加 sanity check：题号必须严格递增 + 数字之间不能有连续空格再数字。

## 2. fengtai（29 题/107 分 = 28+1）

### Q29 真相
- yaml `idx29` stem = `"（1）①AN，BN； 2分 ②； 4分 （2）或． 7分 其它解法请参照评分标准酌情给分"` —— 这是 **真 Q28 的官方答案/评分标准**。
- 源 docx `fengtai_math.docx` 是 **试题+答案合订本**，L73-L90 是试题 Q25-Q28，L181+ 是答案区，L275 = `28．（1）①AN，BN；…`。
- Parser 完整收了 Q1-Q28（→ idx1-idx28），然后又把答案区里的 `28．` 误判成新题，因 idx28 已被占用 → 顺延为 idx29。

### Parser bug 定位
- Parser **没有识别答案分界**（"参考答案" / 加粗 / 分页符 / 重复题号回退）。当源 docx 试题与答案在同一篇中（zxxk "试卷" 版而非 "精品解析"），整段答案会被当成新题继续解析。
- chaoyang/yanshan 用的是 zxxk 切分好的纯试题/纯答案文件（chaoyang/paper_0.docx 是 "【试卷】"），所以没踩；fengtai 是单一合订文件，踩中。

### 跨区共性
**跨区潜在共性**：任何 zxxk 合订 docx 都会触发。chaoyang/yanshan 这次没踩纯属源文件结构差异。建议 parser 加：
1. 检测到 `\d+．` 题号 **回退或重复** 时停止抓题（已抓完 Q28，再出现 Q25/Q28 → 答案区）。
2. 或主动识别 "参考答案" / "评分标准" / "【解析】" 密集段作为终止 marker。

## 3. yanshan（29 题/102 分 = 28+1，但失分性最严重）

### Q29 真相 + 整体错位
- yaml `idx1..idx4` stem 全是 **考生须知** 第 1-5 条（"本试卷共 8 页…/在试卷和答题卡上准确填写…/试题答案一律填涂…/在答题卡上选择题、画图题用 2B 铅笔…"），4 条全部被当成 4 道单选题（type=单选 score=2 各 2 分 = 8 分污染）。
- yaml `idx5` stem = `"考试结束，请将本试卷和答题卡一并交回．第一部分 选择题 第1-8题…"` —— 是须知第 5 条 + section header + **真 Q1 被吞**（"下列图形中，既是轴对称图形又是中心对称图形的是" 不出现在 idx5）。
- yaml `idx6` stem = "若关于x的一元二次方程有实数根，则实数m的取值范围是" = **真 Q5**。
- 也就是说 **真 Q1-Q4 完全丢失**（被须知 + section header 合并吞掉），yaml 实际只有 Q5-Q28 共 24 道真题 + 4 道 noise + 1 个混合块 = 29。

### Parser bug 定位
- 源 docx L2 `考生须知` 后 L3-L7 是 `1．…/2．…/3．…/4．…/5．…` 5 行注意事项，**完全符合 `\d+．` 题头 regex**，被当 5 个单选题吸收。
- 需要 parser **先识别 "考生须知" / "第一部分" 等 section anchor**，在真正的 "一、选择题（共16分…）" anchor 出现之前不收题。chaoyang/fengtai 没踩是因为它们的须知不在主试卷文件中（chaoyang/paper_0.docx 直接 L0=标题 L3=section header，无须知段）。
- **副作用**：真 Q1-Q4 因 idx 已被须知占满，整批被吞 → 严重丢题，**学生练习时这 4 道选择题完全缺失**。

### 跨区共性
yanshan 这种 "考生须知（5 条）在试卷正文开头" 是 zxxk 原卷版常见结构，**11 区里大概率还有 N 个区会踩**（朝阳走 "【试卷】" 切分版避开了，丰台是合订）。强烈建议 parser 必加 "考生须知/注意事项" 段过滤。
