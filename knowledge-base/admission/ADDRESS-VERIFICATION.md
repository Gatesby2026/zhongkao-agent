# 高中校区地址核查与交叉验证方法（SOP）

> 适用：中考志愿系统里**各区高中校区地址**的获取与定稿。朝阳是首个落地区，后续每个区都**照此流程**做。
> 本文是"怎么做"；做出来的数据落在 `<city>/<district>.yaml`（系统读）+ `<city>/<district>_addresses_audit.yaml`（溯源 & 迭代）。

---

## 0. 铁律

1. **涉及孩子升学，地址错会害人 —— 禁止编造。** 拿不准的门牌就标 `low` / `address: null` + `flag`，绝不用猜的门牌冒充确定值。
2. **核查对象 = 统招生实际报到 / 上课的那个校区。** 不是法人注册地、不是"本部"、不是初中部、不是只挂牌不上课的名校分校区。
3. **先定地址，再定坐标。** 坐标层（geocode）的输入必须是本流程核实过的街道门牌，**不能拿校名去地图搜**（见 §1）。

---

## 1. 为什么不能"按校名在地图上搜"

裸校名 geocode 默认落到**本部 / 注册地 / 最有名的那个校区**，对下列情况系统性出错：

- **初高分址**：搜出来是初中部。
- **借址过渡**：新校硬件没完工，高一在老校上课，地图不知道。
- **分年级分校区**：高一、高三不在一个地方。
- **新校启用**：当年迁入新校区，地图 POI 还没更新。

> 朝阳实证：朝阳外国语裸搜 = 慧忠里 216 号本部（**初中**），真高中部在**来广营**，直线差约 **6.2 km**。这就是为什么必须先走本 SOP。

## 2. 信源分级（权威性从高到低）

| 级别 | 信源 | 能定什么 |
|----|----|----|
| **T1** | 北京教育考试院 bjeea.cn《当年招生简章 / 招生计划册》PDF | **唯一**能定"哪个校区收统招 + 专业(班)代码 + 是否分年级"的权威源 |
| **T1** | 区招考中心（朝阳 cyks.org.cn 等）当年招生公告 | 同上，区一级 |
| **T2** | 学校官网（`*.bjchyedu.cn` / 学校独立域名）"联系我们 / 各校区"页 | 高中部**门牌** |
| **T2** | 区政府 / 官媒（bjchy.gov.cn、新华网）新校启用报道 | 定**校区**（常不给门牌） |
| **T3** | 北京中考信息网 zhongkaobj.cn "校情介绍" | 逐校地址，覆盖全但**二手转载** |
| **T4** | 百科 / 地图库 / 上哪学等 | **仅旁证**，不能单独定案 |

定案要求：**至少 2 个独立信源一致**，且其中**校区归属优先取 T1**（门牌可由 T2/T3 补）。

---

## 3. 每校流程

1. **先查 T1 招生简章**：该校今年招几个校区、**哪个校区收统招**、是否分年级。这一步决定"取哪个校区"，错了后面全错。
2. **校区定了再取门牌**：优先 T2 官网联系页，与 T3 校情核对。
3. **交叉**：≥2 独立信源一致 → 门牌可定；冲突按 §5 记录裁决。
4. **留过程数据**（§5）：每个信源说了什么、是否一致、冲突如何裁决。
5. **判定** confidence + action（§4），**写两处**（§6）。

---

## 4. confidence / action 判定

| confidence | action | 条件 |
|----|----|----|
| `high` | `ready` | ≥2 权威源一致 + 高中部明确 + **门牌到位** → 可直接进 geocode |
| `medium` | `verify_doornumber` | 单一权威源，或门牌只到**路名 / 楼号 / 小区**，或多校区高中部靠推断 |
| `low` | `confirm_by_phone` | 报到校区**会随年份变**（借址 / 分年级 / 新校启用），地图无解 → 当年招生简章或电话 |

> 门牌拿不准 → `address: null` + `address_rough`（粗略到路 / 片区），**绝不编门牌号**。

---

## 5. 交叉验证证据格式（过程数据，必须留存）

落在 `<district>_addresses_audit.yaml` 每校的 `evidence` 列表，**逐信源一条**：

```yaml
evidence:
  - source: 学校官网 联系我们
    tier: T2
    url: https://bj80.com/...
    accessed: 2026-06-03
    claim: "高中部 望京北路甲16号"
    agree: yes            # yes | partial | no（与最终采纳是否一致）
  - source: zhongkaobj.cn 校情
    tier: T3
    claim: "望京北路甲16号"
    agree: yes
resolution: 两源一致，采望京北路甲16号；排除白家庄(初中部)。
```

- **冲突案例必须留**（如"地图给的'管庄路63号'是坐标反查描述、非真实门牌，已排除"）——这是最有价值的过程数据，下次别再踩。
- 能抓到的**原始页面 / 截图**存 `_evidence/<district>/<school>/`（gitignore 视体积决定，至少留 URL+日期）。
- 目的：① 溯源（任何一条地址都能查到出处）；② 持续完善（新一轮发现更权威源时，能对比旧证据增量更新）。

---

## 6. 输出落点（两处，缺一不可）

1. **KB 主数据** `<city>/<district>.yaml` —— 每校 `location` 块，系统直接读：
   ```yaml
   location:
     campus: 望京校区（高中本部）
     address: 北京市朝阳区望京北路甲16号
     confidence: high
     action: ready
     flag: ⚠️...        # 仅 medium/low 有风险时写，给系统/用户提示
     source: [bj80.com, zhongkaobj.cn]
     # 多校区：campuses: [{name, address}, ...]
   ```
2. **核查档** `<city>/<district>_addresses_audit.yaml` —— 完整 `evidence` + `resolution` + 本轮局限，溯源 & 迭代用。

## 7. 已知坑清单（持续补充）

- **初高分址**：本部初中地址 ≠ 高中地址（陈经纶本部、八十中）。
- **一校多址**：取**收统招的高中校区**（日坛 / 三里屯 / 十七中 / 东方德才 / 二外附中）。
- **借址过渡**：新校未完工，高一借老校上课（八十中睿德借八十中本部、北京中学科技分校借化工大附中高中部）。
- **分年级分校区**：高一 / 高三不同址（人朝高一高二安华桥+高三太阳宫、清华附中朝阳高一混编）。
- **新校启用**：当年迁入，全网无门牌（北京中学东坝北 2025.8）。
- **挂牌 ≠ 独立校**：莲葩园中学 = 和平街一中北苑校区，非独立办学。
- **"地图描述"冒充门牌**：地图 POI 的"X 路 N 号"可能是坐标反查生成的描述，不是官方门牌。
- **多校区都收统招**：须按校区**分别算距离**（和平街一中：和平街校区 + 北苑莲葩园校区）。

## 8. 本轮（朝阳 2026-06）局限 → 下一区改进

- 研究工具只有 WebSearch（WebFetch 多被拒 / 超时），**bjeea 招生简章 PDF 未逐字读** → 结果 5 校 `low` / 10 校 `medium`。
- evidence 本轮只留到 **source 列表粒度**，未逐字留 `claim/url/accessed` → **下一区按 §5 全量留存**。
- **改进动作**：优先打通 PDF 抓取（参照 memory `skill_gaokzx_curl` 的 curl 直取思路）读 bjeea / 区招考中心当年简章，把校区归属从 T3 升到 T1，可把多数 `medium` 升 `high`。

## 9. 坐标层（地址定稿后做）

地址扎实后，才 geocode。**严禁拿校名去地图搜**（§1），输入必须是核实门牌。

- **脚本**：`scripts/admission/geocode_addresses.py --district <pinyin>`
  - 读 `<district>.yaml` 每校 `location.address`（+ `campuses[].address`；address 为 null 时退 `address_rough` 并标 `approx`）。
  - 高德 geocode（**GCJ-02**，与路网 direction API、前端高德底图同坐标系，直接用）。
  - **名称交叉校验**：另用高德 POI 名称搜索取校名坐标，与地址 geocode 点算直线距离 `name_xcheck_km`。**坐标永远以核实地址为准**；gap 大只作 flag（对已纠正/多校区/借址校，大 gap 是预期：如朝阳外国语地址=来广营 vs 校名POI=慧忠里本部 6km，恰恰印证地址修对了）。
  - 产出 `<district>_coords.json`（派生可重生，入库供部署）。
- **geo_status**：`ok`（落本区+level精细门址/POI/道路）/ `coarse`（片区级 住宅区/村庄/小巷，仍可算通勤）/ `review`（不落本区，需人工）/ `failed`。
- **消费**：`distance.py` 的 `load_kb_coords()` 优先读这份坐标，旧坐标库（highschools_final.json 等裸名 geocode 产物）仅兜底。
- **朝阳首轮**：27 校 22 ok / 6 coarse / 0 review / 0 failed；多校区分别出坐标。
- **重生**：地址改了就重跑脚本覆盖 json，无需手改坐标。原始 API 落 `.cache/addr_geocode_cache.json`（gitignore）。
