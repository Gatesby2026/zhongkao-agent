# 中考学习规划智能 Agent — 产品设计文档

> 最后更新：2026-04-11
> 状态：方案设计阶段，持续演进中

---

## 一、产品定位

面向中考考生的全生命周期智能学习规划系统。

核心能力：
- 根据考生各科实际水平，精准诊断薄弱环节（到知识点级别）
- 结合所在地区/区县的考试政策、考纲、题型差异，制定个性化目标
- 给出**可执行的学习路径**：学什么、用什么学、怎么学、学到什么程度算过关
- 全程跟踪，动态调整——从入驻到中考结束

**不是**一个通用的学习计划生成器，而是一个**深度垂直的中考备考专家系统**。

---

## 二、核心设计理念

### 2.1 边际收益最大化

中考提分的本质是把有限时间花在"投入产出比"最高的科目和知识点上。

关键原则：
- **低分科提分容易，高分科提分难** — 数学从 80 提到 100 比从 110 提到 130 容易得多
- **各科有"天花板效应"** — 短期内从 140 提到 150 极难，不值得大量投入
- **要设"保底线"和"冲刺线"** — 每科给一个稳妥目标和理想目标

边际收益递减模型：
```
每科的"提分难度系数" = f(当前分数 / 满分比例)
时间分配权重 ∝ 预期提分空间 / 提分难度系数
```

### 2.2 知识库驱动，而非纯 LLM 生成

- 结构化数据（政策/分值/考点/学习路径）→ 确定性计算，保证准确
- LLM 负责个性化表达、学习建议润色、鼓励话术 → 保证体验
- 两者结合 → 精准且有深度

## 四、知识库详细设计（四层结构）

### 第1层：地区政策库

存储各地中考的基本配置信息。

```yaml
# 示例：北京市 2026 中考配置
region: 北京市
districts:
  - name: 朝阳区
    notes: "近年侧重综合实践类题目"
  - name: 海淀区
    notes: "难度偏高，区分度大"
  - name: 西城区
    notes: "题目规范，紧扣考纲"
```

数据来源：各地教育局公开文件，LLM 辅助提取结构化信息。

### 第2层：学科考纲库

各科的分值、题型、考点权重、特殊规则。

```yaml
# 示例：北京中考·数学
subjects:
  - name: 语文
    full_score: 100
    exam_type: 闭卷
    duration: 150min
    key_modules:
      - name: 古诗文默写
        score: 10
        difficulty: 低
        strategy: "纯记忆，必拿分"
      - name: 作文
        score: 40
        difficulty: 中
        strategy: "准备3-4个万能素材框架"
    score_band_strategy:
      "0-60": "基础严重薄弱，先抓默写+阅读理解基本分"
      "60-80": "作文和阅读理解是突破口"
      "80-90": "精细化训练，减少非智力失分"
      "90+": "保持即可，时间让给其他科"

  - name: 道德与法治
    full_score: 80
    exam_type: 开卷
    duration: 与历史合卷90min
    special_rules: "可带教材和笔记进考场"
    key_modules:
      - name: 时政热点
        score: ~20
        strategy: "开卷科的关键不是记忆，是快速检索能力和答题框架"
    score_band_strategy:
      "0-50": "整理教材目录索引，练习快速翻书定位"
      "50-65": "强化答题模板，练习材料分析题"
      "65+": "刷真题找答题节奏，控制翻书时间"
```

### 第3层：诊断标准库

对每个知识模块定义水平分级，用于精准定位考生在每个模块的水平。

```yaml
module: 函数（一次/反比例/二次）
subject: 数学
region: 北京

diagnostic:
  entry_test: "10道基础判定题，5分钟完成"
  levels:
    - id: L0_零基础
      signal: "看到 y=kx+b 不知道k和b代表什么"
    - id: L1_概念模糊
      signal: "知道公式但不会画图，或画图但不会读图"
    - id: L2_基础会做
      signal: "能做课本例题，但模考的变式题/综合题丢分"
    - id: L3_熟练应用
      signal: "函数部分基本不丢分，偶尔综合题失误"
```

### 第4层：学习路径库（核心价值层）

每个知识模块 × 每个水平等级 = 一条具体的学习路径。

路径包含：
- **学什么**（topic）
- **用什么学**（resources：具体到教材哪一章、参考书哪个专题、哪个平台的视频）
- **怎么学**（how_to_use：不是"买本五三"，而是"五三第三章只做★★题，每天4题限时20分钟"）
- **学到什么程度算过关**（pass_criteria：可量化的判定标准）
- **容易踩的坑**（common_pitfalls：提前预警）

```yaml
learning_paths:

  L0_零基础:
    target: 升到 L2
    estimated_hours: 20-25h
    steps:
      - order: 1
        topic: "一次函数的概念与图像"
        goal: "能独立画出y=2x+1的图像，说出k和b的几何意义"
        resources:
          - type: 教材
            name: "北师大版九上 第X章"
            how_to_use: "先看例题，抄一遍解题过程，再合上书做课后1-5题"
          - type: 视频
            name: "国家中小学智慧教育平台 - 一次函数"
            how_to_use: "每个视频看完后暂停，自己复述核心结论"
          - type: 练习
            name: "五年中考三年模拟 - 一次函数基础篇"
            how_to_use: "只做★和★★题，跳过★★★"
        pass_criteria: "连续做对10道一次函数基础题，无计算错误"
        common_pitfalls:
          - "把 y=kx+b 的b看成斜率 → 反复默写：k是斜率，b是截距"
          - "画图只描两个点 → 强制要求描三个点验证"

      - order: 2
        topic: "一次函数的应用（行程/费用问题）"
        goal: "能从文字题中提取k和b，列出函数关系式"
        resources:
          - type: 教材
            name: "课本应用题例题"
            how_to_use: "重点练习'读题→找变量→列式'这个过程"
          - type: 练习
            name: "中考真题分类 - 一次函数应用题（近3年北京卷）"
            how_to_use: "先自己做，错题整理到本子上，标注错因"
        pass_criteria: "北京近3年一次函数应用题能做对80%"
        common_pitfalls:
          - "不会设变量 → 固定套路：时间设t，路程设s，费用设y"

  L1_概念模糊:
    target: 升到 L2
    estimated_hours: 12-15h
    steps:
      - order: 1
        topic: "图像与性质速通"
        goal: "看到图像能写出解析式，看到解析式能画出图像"
        resources:
          - type: 专项
            name: "自制：函数图像速查卡"
            how_to_use: "正面画图，背面写性质，每天翻5分钟"

  L2_基础会做:
    target: 升到 L3
    estimated_hours: 8-10h
    skip: "跳过基础概念，直接进入变式训练"
    steps:
      - order: 1
        topic: "中考真题函数专项刷题"
        goal: "熟悉北京中考函数题的出题套路"
        resources:
          - type: 真题
            name: "北京近5年中考数学 - 函数相关题汇编"
            how_to_use: "限时训练，每组4题20分钟"
        pass_criteria: "正确率稳定在90%以上"
```

#### 资源推荐原则

| 原则 | 说明 |
|------|------|
| 只推荐主流验证资源 | 课本、五三、国家智慧教育平台、各区一模二模真题 |
| 不推荐书名，推荐用法 | "五三第三章★★题，每天4题限时20分钟" |
| 资源匹配学生水平 | L0 学生不做压轴题，L3 学生不刷基础题 |
| 谨慎推荐网课/自媒体 | 需确认后再纳入 |

## 五、考生画像与诊断

### 5.1 输入数据

- **基本信息**：所在城市、区县、学校、年级
- **各科当前成绩**：最近 2-3 次模考分数
- **目标**：目标学校/分数线（可选）
- **可用时间**：每天可用学习时间
- **细粒度数据（可选）**：各科各题得分，映射到知识点维度

### 5.2 诊断输出示例

```
考生小明 - 数学诊断
━━━━━━━━━━━━━━━━━━━━━
总分: 95/120

知识模块         得分率    判定      建议
──────────────────────────────────────
数与代数          88%     ✓ 良好    保持
方程与不等式       92%     ✓ 优秀    保持
函数              65%     ✗ 薄弱    重点突破 ← 投入产出比最高
几何-基础          90%     ✓ 优秀    保持
几何-综合题        40%     ✗ 薄弱    适度投入（难度大，收益不确定）
统计与概率         95%     ✓ 优秀    保持

→ 建议：函数板块是你数学提分的关键突破口
→ 预计投入 15 小时专项训练，可提 8-12 分
→ 几何综合作为选做拔高，不建议投入过多时间
```

### 5.3 学习路径输出示例

```
📊 你的数学·函数板块诊断结果
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

当前水平：L1（概念模糊）
目标水平：L2（基础会做）→ 预计提 8-12 分
预计需要：12-15 小时（约 2 周，每天 1 小时）

▎第1步（约5小时）
  图像与性质速通
  ├ 用什么：北师大版课本第X章 + 函数图像速查卡
  ├ 怎么学：正面画图背面写性质，每天翻卡5分钟
  ├ 过关标准：看到图说出解析式，看到式画出图，连续10题不错
  └ 易踩的坑：反比例函数的k>0和k<0图像位置容易混

▎第2步（约4小时）
  应用题列式训练
  ├ 用什么：五三·一次函数应用篇 ★★题
  ├ 怎么学：读题→找变量→列式，每题标注"谁是x谁是y"
  ├ 过关标准：北京近3年应用题做对80%
  └ 易踩的坑：忘记写定义域/自变量范围

▎第3步（约4小时）
  中考真题专项
  ├ 用什么：北京近5年函数真题汇编
  ├ 怎么学：限时训练，错题本记录错因
  └ 过关标准：选择填空函数题正确率90%

完成这3步后会重新测试，确认进入 L2
```

## 六、全生命周期设计

### 6.1 时间轴（以初三为例）

```
[入驻评估]        [月度复盘]           [考前冲刺]
    │                 │                    │
    ▼                 ▼                    ▼

9月 ──── 12月 ──── 3月(一模) ──── 5月(二模) ──── 6月中考
 │         │          │              │            │
 基础摸底   阶段检测    一模定位        二模微调      最终策略
 建立画像   调整计划    重新分配资源     稳定心态      查漏补缺
```

### 6.2 各阶段 Agent 行为

| 阶段 | Agent 核心任务 | 关键输入 |
|------|--------------|---------|
| 入驻 | 全科诊断，建立基线画像 | 近期成绩、自评、目标校 |
| 常规期 | 周计划生成，时间分配优化 | 每周学习记录、小测成绩 |
| 模考后 | 深度分析，策略大调整 | 模考各科详细成绩（最好到小题） |
| 冲刺期 | 保强补弱，心态管理 | 二模成绩、目标校分数线 |

## 七、技术方案

### 7.1 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 前端 | Next.js | SSR + 移动端适配 |
| 后端 | Node.js 或 Python | 核心算法逻辑不复杂 |
| 数据库 | PostgreSQL | 考生数据、学习记录、成绩追踪 |
| 知识库 | 结构化 YAML/JSON 文件 | 按地区/学科组织，版本管理 |
| LLM | Claude API | 生成个性化分析报告和学习建议 |

### 7.2 数据模型核心关系

```
考生 → 所属地区 → 关联该地区考试配置
考生 → 多次成绩记录 → 每次关联到知识点得分
考生 → 学习计划 → 按周拆解 → 每日任务
```

## 八、项目难点与工作量评估

| 难点 | 占比 | 说明 |
|------|------|------|
| 知识库质量 | 60% | 各地政策、考纲、分值、题型、学习路径。不准则整个产品无价值 |
| 策略算法 | 20% | 时间分配、目标设定、阶段规划逻辑 |
| 产品体验 | 15% | 考生愿意持续使用、持续输入数据 |
| 技术实现 | 5% | 最简单的部分 |

**第4层学习路径库不可能纯靠技术完成**，必须有经验丰富的中考老师参与设计骨架，LLM 负责扩展和个性化表述。

## 九、推进计划

### 阶段一：MVP 验证（建议先做）

- 选定 1 个城市（建议北京，公开资料多）
- 选定 3 个核心科目（数学、英语、物理）
- 四层知识库全部做通
- 找老师验证知识库准确性
- 做出可用的 MVP 给真实考生试用

### 阶段二：产品化

- 补全该城市剩余科目
- 完善全生命周期追踪功能
- 优化产品体验

### 阶段三：扩展

- 扩展到更多城市/地区
- 知识库扩展（模式已验证，边际成本递减）
