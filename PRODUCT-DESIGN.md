# 中考智能学习规划 — 产品整体设计

> 版本：v2.0 | 更新日期：2026-04-12
> 定位：面向北京中考生及家长的 AI 个性化学习规划系统

---

## 一、产品定位重审

### 核心理念

产品应该像一位**经验丰富的班主任**：

1. **先充分了解学生** — 通过测评、刷题、对话，建立精准画像
2. **再给出针对性建议** — 不是通用模板，是"你，此刻，该做什么"
3. **持续跟踪调整** — 随着学习进展动态更新计划

### 三层能力模型

```
┌─────────────────────────────────────────────────────────┐
│                  产品核心能力                              │
├──────────────┬───────────────────┬──────────────────────┤
│  能力一       │  能力二             │  能力三                │
│  查漏补缺     │  单科规划            │  全科规划               │
│  (聚焦模块)    │  (科目完整覆盖)       │  (中考全局统筹)          │
├──────────────┼───────────────────┼──────────────────────┤
│  针对薄弱模块   │  数学全部 7 模块      │  数学+语文+英语          │
│  制定专项训练   │  从诊断到计划到执行    │  +物理+化学+道法/历史     │
│  1-2 周见效    │  覆盖中考全范围       │  跨科时间分配优化         │
│  适合明确短板   │  适合数学整体提升     │  适合全面备考规划         │
├──────────────┼───────────────────┼──────────────────────┤
│  ✅ 已实现     │  ⚠️ 部分实现         │  🔲 待实现              │
│  /drill       │  /plan (需升级)      │  /plan-all (新建)       │
└──────────────┴───────────────────┴──────────────────────┘
```

### 前提：充分了解学生

三层能力都依赖于对学生的精准了解，这是个性化的基础：

```
了解学生的手段：
  ① 快速测评（5 分钟 10 题）     → 7 模块初步诊断          ✅ 已实现
  ② 分数+目标校                  → 差距分析               ✅ 已实现
  ③ 刷题行为数据                 → 动态修正模块水平         🔲 待接入
  ④ LLM 对话诊断                → 精准到知识点的错因分析    🔲 Phase 2
  ⑤ 模考/真题成绩导入             → 真实数据校准            🔲 Phase 3

了解的深度 → 决定计划的精准度
```

### 3.1 能力一：查漏补缺 /drill

**定位**：已知薄弱模块 → 1-2 周专项突破计划

**现状**：✅ 已基本实现，需小幅优化

```
入口：首页"查漏补缺"卡片 / 测评结果页"先练最弱模块" / 画像页"去练习"

流程：
  Step 1: 选择薄弱模块（7 选 1）
          → 如已有画像数据，自动高亮推荐最薄弱的模块
          → 显示当前 level + confidence

  Step 2: 确认训练参数
          → 当前水平（从画像读取 or 手动选）
          → 每周训练时间（3/6/9/12 小时）
          → 具体薄弱点（可选，如"二次函数图象变换"）

  Step 3: AI 生成专项计划
          → SSE 流式输出
          → 包含：目标（升 1 级）、每日任务、推荐题目、注意事项
          → 从知识库匹配真题/模考题

  Step 4: 计划执行（🔲 待实现）
          → 每日打卡
          → 刷题结果写入画像
          → 周末复盘（哪些提升了、哪些还需加强）
```

**待优化项**：
- [ ] 从画像自动推荐最弱模块（当前需手动选）
- [ ] 刷题结果回写画像（`updateModuleFromDrill`）
- [ ] 计划保存到 `plan_records` 表
- [ ] 完成后引导：做完这个模块 → 推荐下一个薄弱模块

### 3.2 能力二：数学完整规划 /plan/math

**定位**：覆盖中考数学全部 7 个模块，从诊断到计划到执行

**现状**：⚠️ 部分实现（当前 /plan 只做诊断+计划，缺执行跟踪和阶段拆分）

**核心升级**：从"生成一份计划"升级为"诊断-规划-执行-复盘"完整闭环

```
┌─────────────────────────────────────────────────────────┐
│              数学完整规划 — 四步闭环                        │
├──────────┬──────────┬───────────┬──────────────────────┤
│ Step 1   │ Step 2   │ Step 3    │ Step 4               │
│ 诊断定位  │ 生成规划  │ 执行计划   │ 阶段复盘              │
├──────────┼──────────┼───────────┼──────────────────────┤
│          │          │           │                      │
│ 测评 or   │ AI 生成   │ 每日任务   │ 每周/每月复盘         │
│ 画像读取   │ 整体路线  │ 打卡追踪   │ 重新测评             │
│          │          │           │ 对比进步             │
│ →7模块水平 │ →阶段拆分  │ →刷题记录  │ →调整计划             │
│ →差距分析  │ →周计划    │ →画像更新  │ →下阶段规划           │
│          │          │           │                      │
│ ✅ 已实现  │ ⚠️ 需升级 │ 🔲 待实现  │ 🔲 待实现             │
└──────────┴──────────┴───────────┴──────────────────────┘
```

#### Step 1: 诊断定位（✅ 已实现）

```
入口方式（三选一，系统自动判断）：

  A. 已有画像（confidence ≥ 0.5 的模块 ≥ 5 个）
     → 直接读取画像数据，跳到 Step 2
     → 页面顶部显示"基于你的测评/刷题数据诊断"

  B. 刚做完测评（URL 带 fromAssessment=1）
     → 读取测评结果，直接进入 Step 2
     → 页面显示"基于本次测评结果"

  C. 没有数据
     → 显示简化表单：区域 + 分数 + 目标校
     → 模块自评（默认全"不确定"）
     → 引导语："不确定没关系，先做个初步计划，后续做测评会更精准"
```

#### Step 2: 生成规划（⚠️ 需升级）

当前已有基础版，需增加：

```
现有输出：
  ✅ 诊断卡（当前分→目标分→差距→周数）
  ✅ 模块表（水平、预估分、提升空间、时间分配、优先级）
  ✅ AI 生成的整体路线 + 周计划

需增加的输出：
  🔲 阶段拆分：把剩余时间分为 2-4 个阶段
     ├── 阶段一（前 1/3 时间）：基础补漏 — 聚焦 L0/L1 模块
     ├── 阶段二（中间 1/3）：专项提升 — 聚焦 L1→L2 的模块
     └── 阶段三（后 1/3）：综合冲刺 — 压轴题+模拟考+查缺补漏

  🔲 本周具体任务清单：
     ├── 周一：函数 - 一次函数图象（45min）
     ├── 周二：方程 - 分式方程验根（45min）
     ├── ...
     └── 每个任务关联具体知识点 + 推荐题目来源

  🔲 预期里程碑：
     ├── 第 2 周末：L0 模块全部升至 L1
     ├── 第 4 周末：总分预期提升 8-10 分
     └── 中考前 2 周：全模块达到 L2+
```

#### Step 4: 阶段复盘（🔲 待实现）

```

## 四、用户旅程（典型路径）

### 路径 A：焦虑家长（最常见）

```
第 1 天（获客）：
  搜索"中考分数线" → 打开首页 → 点"分数查校"
  → 输入 朝阳区 78 分 → 看到三档学校匹配
  → 心理活动："哦，78分只能上工大附中，差八十中还有12分"
  → 注册登录（手机号验证码）

第 1 天（转化）：
  → 点"快速测评" → 做 10 题（可能是家长替孩子做，也可能发给孩子）
  → 看到诊断结果："函数L0，方程L1，圆L0..."
  → 点"生成数学完整规划"
  → 看到分阶段提分方案 + 本周任务
  → 截图发到家长群 / 发给孩子

第 2-7 天（留存）：
  → 让孩子按计划每天做 45 分钟
  → 孩子做刷题 → 画像自动更新
  → 周末看进度报告

第 30 天（复购/续用）：

## 六、数据闭环设计

所有功能通过学生画像串联，形成数据飞轮：

```
                    ┌─────────────┐
                    │   学生画像    │
                    │  (Profile)   │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ 测评     │ │ 刷题     │ │ 对话诊断  │
        │ 写入画像  │ │ 写入画像  │ │ 写入画像  │  ← 数据输入
        └────┬─────┘ └────┬─────┘ └────┬─────┘
             │            │            │
             ▼            ▼            ▼
        ┌──────────────────────────────────┐
        │           画像更精准               │  ← 飞轮核心
        └──────────────┬───────────────────┘
                       │
              ┌────────┼────────┐
              ▼        ▼        ▼
        ┌────────┐┌────────┐┌────────┐
        │查漏补缺 ││数学规划 ││全科规划 │       ← 计划输出
        │更精准   ││更精准   ││更精准   │
        └───┬────┘└───┬────┘└───┬────┘
            │         │         │
            └─────────┼─────────┘
                      ▼
               ┌────────────┐
               │ 执行 → 刷题  │               ← 回到输入
               │  → 画像更新  │
               └────────────┘
```

### 关键数据流

| 触发事件 | 数据写入 | 画像影响 |
|---------|---------|---------|
| 完成测评 | `assessment_records` | 7 模块 level 更新，confidence = 0.6 |
| 刷题完成 | `drill_records` | 对应模块 level EMA 更新，confidence += 0.1 |
| 生成计划 | `plan_records` | targetSchool、hoursPerDay 更新 |
| 分数查校 | — | district、currentScore 更新 |
| 编辑画像 | `profiles` | 直接更新 |
| 每日打卡 | `drill_records` | 行为数据积累，streak 更新 |

## 八、设计原则

1. **数学先行**：数学有完整题库和 AI 诊断能力，做到极致再拓展其他科
2. **被动采集**：用户操作自然产生数据，不做问卷填表
3. **渐进展示**：根据画像完整度决定展示什么，不让新用户看到空白页
4. **移动优先**：初三学生和家长主要用手机，所有页面移动端优先设计
5. **结果可信**：所有 AI 生成的计划都标注数据来源，让用户知道"为什么这么建议"
6. **闭环为王**：诊断→计划→执行→复盘，不断循环，画像越来越准，计划越来越精确
