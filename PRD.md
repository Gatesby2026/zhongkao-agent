# 中考智能学习规划 — 产品需求文档（PRD）

> 版本：v3.0 | 更新日期：2026-04-12
> 产品定位：面向北京中考生家长的 AI 学习顾问
> 核心原则：**产品的每一个页面都在回答家长的一个具体问题**

---

## 一、用户与场景

### 1.1 目标用户

| 用户 | 画像 | 核心诉求 | 使用场景 |
|------|------|---------|---------|
| **主用户：家长** | 初三学生的父母，对中考政策一知半解，焦虑但不知如何帮 | "孩子能上什么学校？差多少？怎么补？" | 晚上看手机研究，周末和孩子讨论 |
| **执行用户：学生** | 初三学生，被家长安排或自驱 | "今天具体做什么题？做多久？" | 坐在书桌前执行 |

> **设计优先级：家长 > 学生。** 家长是决策者、付费者、传播者。学生是执行者。

### 1.2 家长五问（产品核心场景）

家长使用本产品的心理路径是五个递进的问题。产品的全部功能都是为了回答这五个问题。

| # | 家长的问题 | 翻译成产品需求 | 对应功能 |
|---|-----------|--------------|---------|
| Q1 | "我孩子 395 分，能上八十中吗？差多少？" | 总分 → 学校匹配 + 差距量化 | 分数查校（总分模式） |
| Q2 | "差的这些分，从哪里补最有效？要多久？" | 各科提分性价比 + 时间预估 | 诊断解读 + 提分方案 |
| Q3 | "具体每天做什么？怎么执行？" | 精确到天的学习任务 | 每日任务 + 学科详细计划 |
| Q4 | "这周做了没？做得怎么样？进步了没？" | 执行追踪 + 进步可视化 | 周报 + 进度仪表盘 |
| Q5 | "这个方案靠谱吗？凭什么信你？" | 数据透明 + 案例佐证 + 专业背书 | 信任体系 |

> **产品路线图：先把 Q1-Q2 回答到位，再逐步补齐 Q3-Q5。**

### 1.3 家长 vs 学生视角

产品不做两套完整界面，但同一功能的呈现重点不同：

| 维度 | 家长视角 | 学生视角 |
|------|---------|---------|
| 首页 | 概览仪表盘：总分趋势、学校差距、本周完成率 | 今日任务 + 刷题入口 |
| 诊断结果 | "函数基础不牢，这部分中考占 15-20 分，补上预计涨 8-12 分" | "函数 L0，需要从一次函数图象开始补" |
| 计划 | 总览：各科时间分配 + 预期提分 + 关键里程碑 | 具体：今天做什么题、做多久、做完了没 |
| 数据指标 | 总分变化、目标校差距、进步趋势 | 正确率、完成率、错题数 |
| 使用频率 | 每周 1-2 次 | 每天 |

**实现方式**：注册时选择角色（家长/学生），首页和关键页面根据角色显示不同重点。

## 二、考试基础

### 2.1 2026 北京中考方案

**总分 510 分，6 门计分科目**（2025 年改革后方案）：

| 科目 | 满分 | 构成 | 备注 |
|------|------|------|------|
| 语文 | 100 | 笔试 100 | |
| 数学 | 100 | 笔试 100 | |
| 英语 | 100 | 笔试 60 + 听说机考 40 | 听说两次机会取高分 |
| 物理 | 80 | 笔试 70 + 实验操作 10 | |
| 道德与法治 | 80 | 笔试 70（开卷）+ 综合素质评价 10 | |
| 体育与健康 | 50 | 现场考试 30 + 过程性考核 20 | |

- **无选考科目**。历史、地理、化学、生物为考查科目（等级 A/B/C/D），不计入总分。
- 文化课考试时间：6 月 24-25 日。

### 2.2 各科特点（影响产品策略）

| 科目 | 提分特点 | 产品策略 |
|------|---------|---------|
| 数学 | 模块化强，差距大，提分空间大 | **主力科目**：已建题库 + AI 诊断，给精确计划 |
| 英语 | 听说 40 分可突破，笔试阅读/完形可专项 | 听说机考 + 阅读专项，结构化程度高 |
| 物理 | 公式+实验，有规律可循 | 公式掌握 + 实验操作备考 |
| 道法 | 开卷考试，答题框架 > 死记硬背 | 框架整理 + 材料分析方法 |
| 语文 | 高分段提分难，作文主观性强 | 维持为主，作文素材积累 |
| 体育 | 过程性 20 分基本固定，现场 30 分可训练 | 体能训练建议，非产品核心 |

## 三、产品流程（顾问式引导）

### 3.1 核心理念：顾问，不是工具箱

> **工具箱**："这里有测评、规划、刷题三个功能，你来选。"
> **顾问**："你孩子数学 78 分想上八十中？差 12 分。我看了情况，函数和圆最弱，补上这两块能涨 10-15 分。按这个方案走，4 周后再看。"

**产品不让用户选择路径，而是根据用户状态自动引导。**

### 3.2 智能引导逻辑

```
新用户（无数据）
  → 首页只有一个入口："3 分钟了解孩子的真实水平"
  → 做测评（或直接输入各科成绩）
  → 自动进入「诊断解读」
  → 解读完毕 → 自动生成「提分方案」
  → 方案包含：全科时间分配 + 数学详细计划 + 其他科方向建议

老用户（有画像）
  → 首页直接显示「本周重点」
  → 家长：概览仪表盘（总分差距 + 各科进度 + 本周完成率）
  → 学生：今日任务列表

考前冲刺（< 30 天）
  → 自动推荐冲刺方案
  → 聚焦高性价比科目和模块

模考后
  → 引导导入模考成绩
  → 自动更新画像 + 重新生成方案
```

### 3.3 主流程（家长视角）

```
Step 1: 了解现状        Step 2: 诊断解读        Step 3: 制定方案
┌──────────────┐     ┌──────────────┐      ┌──────────────┐
│ 输入各科成绩   │     │ 通俗解读诊断   │      │ 全科时间分配   │
│ + 目标学校     │ ──→ │ + 提分预期     │ ──→  │ + 数学详细计划  │
│ + 数学测评     │     │ + 信心数据     │      │ + 预期提分承诺  │
└──────────────┘     └──────────────┘      └──────────────┘
                                                  │
Step 5: 看到进步        Step 4: 跟踪执行         │
┌──────────────┐     ┌──────────────┐          │
│ 周报/月报     │     │ 每日任务      │          │
│ 进步可视化    │ ←── │ 执行打卡      │  ←───────┘
│ 调整方案      │     │ 画像更新      │
└──────────────┘     └──────────────┘
```

## 四、功能详细设计

### 4.1 分数查校（回答 Q1：能上什么学校？差多少？）

**路径**：`/score-check`
**状态**：⚠️ 需升级（当前仅支持数学单科）

#### 4.1.1 问题

家长关心的是：**"我孩子总分 395，能上什么学校？"** 而不是"数学 78 分能上什么学校"。中考录取线按总分 510 划定。

#### 4.1.2 设计

支持两种模式：

**快速模式（默认）**：
```
┌──────────────────────────────────────┐
│ 🏫 我的分数能上什么学校？              │
│                                      │
│  所在区：[朝阳区 ▾]                   │
│  当前总分：[395] / 510               │
│                                      │
│  [查看匹配学校 →]                     │
└──────────────────────────────────────┘
```

**详细模式（展开）**：
```
│  各科成绩：                           │
│  语文 [82]/100  数学 [78]/100        │
│  英语 [75]/100  物理 [55]/80         │
│  道法 [65]/80   体育 [40]/50         │
│  ──────────────────                  │
│  总分：395 / 510   自动计算           │
```

**输出结果**：
```
┌──────────────────────────────────────┐
│ 你的当前总分：395 / 510               │
│                                      │
│ 🎯 冲刺校（差 45-55 分）              │
│    八十中   参考线 450    差 55 分     │
│    陈经纶   参考线 445    差 50 分     │
│                                      │
│ ✅ 稳妥校（差 15-25 分）              │
│    和平街一中  参考线 415  差 20 分    │
│                                      │
│ 🛡️ 保底校（高于参考线）               │
│    三里屯一中  参考线 385  高 10 分    │
│                                      │
│ ───────────────────────              │
│ 💡 想冲刺八十中？差 55 分。            │
│    做个快速测评，3 分钟看看哪里提分最快 │
│    [开始测评 →]                       │
└──────────────────────────────────────┘
```

**关键设计点**：
- 默认输入总分，不要求拆科（降低门槛）
- 详细模式可选，输入各科后自动算总分
- 结果页直接显示"差 X 分"，量化差距
- 底部引导到测评，形成转化漏斗

### 4.2 诊断解读（回答 Q2：从哪里补最有效？要多久？）

**路径**：`/diagnosis`
**状态**：🔲 新增（评审指出的关键缺失环节）

#### 4.2.1 为什么需要这个页面

当前产品从"测评结果"直接跳到"生成计划"，跳过了**让家长理解和建立信任**的过程。

家长看到"函数 L0"的反应不是"好，我知道了"，而是：
- "L0 是什么意思？严重吗？"
- "其他孩子都什么水平？我孩子算差的吗？"
- "这个 L0 提上去大概能涨多少分？"
- "为什么你说是 L0？依据是什么？"

#### 4.2.2 设计

诊断解读页在「测评/成绩输入」和「生成方案」之间，是建立信任的关键环节。

每个方案必须包含量化的预期产出：

```

```

## 七、数据闭环

### 7.1 画像驱动一切

所有功能通过学生画像串联。画像越完整，方案越精准。

```
数据输入 ──────────────────→ 学生画像 ──────────────────→ 方案输出
                               │
测评（10题）    → 7模块level    │   诊断解读（通俗语言）
各科成绩输入   → 总分+各科     │   全科时间分配
刷题行为       → 动态修正      │   数学详细计划
模考成绩导入   → 真实校准      │   每日任务
                               │   预期提分承诺
                               │
                       ← 执行数据回流 ←
```

### 7.2 关键数据流

| 触发事件 | 写入 | 画像影响 |
|---------|------|---------|
| 各科成绩输入 | profiles | 总分 + 各科分数 + 学校匹配更新 |
| 完成测评 | assessment_records | 7 模块 level 更新，confidence = 0.6 |
| 刷题完成 | drill_records | 对应模块 level EMA 更新，confidence += 0.1 |
| 模考成绩导入 | profiles + assessment_records | 全面校准，confidence = 0.9 |
| 生成计划 | plan_records | 目标校、每日时间 |
| 每日打卡 | drill_records | streak、完成率 |

### 7.3 画像完整度（面向用户）

不再显示百分比数字，改为分级提示：

| 内部分值 | 用户看到的 | 触发条件 |
|---------|----------|---------|
| 0-20% | "我们还不够了解你的孩子，做个测评吧" | 仅注册，无数据 |
| 20-50% | "有了初步了解，可以给出大方向的建议" | 有成绩输入或测评 |
| 50-75% | "了解较充分，方案精确度较高" | 测评 + 部分刷题 |
| 75%+ | "了解充分，方案精确度很高" | 测评 + 持续刷题 + 模考 |

## 九、设计原则

1. **顾问思维**：每个页面回答家长的一个具体问题，不是展示系统的一个功能
2. **结果承诺**：每个方案必须有量化的预期产出和置信条件
3. **通俗表达**：不出现 L0/L1/L2 等技术标签，用"基础薄弱/基本掌握/掌握良好"
4. **智能引导**：不让用户选择路径，根据状态自动推荐下一步
5. **数学先行**：数学有完整能力（题库+测评+AI），做到极致再拓展
6. **被动采集**：用户操作自然产生数据，不做问卷填表
7. **移动优先**：初三学生和家长主要用手机
8. **闭环为王**：了解→诊断→方案→执行→反馈，不断循环
9. **信任优先**：宁可少给建议，不给没有依据的建议
10. **紧迫感**：距中考 73 天，每个功能都要问"这对当前用户有用吗"
