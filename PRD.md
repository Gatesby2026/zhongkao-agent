# 中考智能学习规划 — 产品需求文档（PRD）

> 版本：v4.0 | 更新日期：2026-05-10
> 产品定位：面向北京中考生家长的 AI 私人教研组
> 核心原则：**通过微信对话，持续陪伴学生备考全程**
> 交付形态：微信群组（正式版）/ 微信私聊（MVP）

---

## 一、产品定位

### 1.1 一句话描述

家长扫码，在微信里拥有一个专属于孩子的"私人教研组"——班主任和各科老师通过群聊持续跟进、主动推送、随时答疑。

### 1.2 与传统产品的本质区别

| 维度 | 传统学习 App | 本产品 |
|------|------------|--------|
| 入口 | 下载 App / 打开网页 | 微信群（零门槛） |
| 交互 | 用户主动操作 | 老师主动触达 |
| 建档 | 填写问卷表单 | 对话中自然收集 |
| 感受 | 使用工具 | 有老师在跟着 |
| 留存 | 靠用户习惯 | 靠老师推送触达 |

### 1.3 家长核心诉求（五问）

产品所有对话设计都围绕回答这五个问题：

| # | 家长的问题 | 产品如何回答 |
|---|-----------|------------|
| Q1 | "我孩子 395 分，能上八十中吗？差多少？" | 班主任收集成绩后直接给出学校匹配和差距 |
| Q2 | "差的这些分，从哪里补最有效？" | 班主任给出各科提分性价比分析和方案 |
| Q3 | "具体每天做什么？" | 班主任每日推送学习任务 |
| Q4 | "这周做了没？进步了没？" | 班主任每周推送学习报告 |
| Q5 | "这个方案靠谱吗？" | 各科老师专业解答，数据支撑 |

## 二、考试基础

### 2.1 2026 北京中考方案

**总分 510 分，6 门计分科目**（2025 年改革后方案）：

| 科目 | 满分 | 构成 | 备注 |
|------|------|------|------|
| 语文 | 100 | 笔试 100 | 闭卷，150分钟 |
| 数学 | 100 | 笔试 100 | 闭卷，120分钟 |
| 英语 | 100 | 笔试 60 + 听说机考 40 | 听说两次机会取高分 |
| 物理 | 80 | 笔试 70 + 实验操作 10 | 闭卷，70分钟 |
| 道德与法治 | 80 | 笔试 70（开卷）+ 综合素质评价 10 | 开卷，70分钟 |
| 体育与健康 | 50 | 现场考试 30 + 过程性考核 20 | |

- **考查科目**（不计入总分，仅评等级 A/B/C/D）：历史、地理、化学、生物
- 文化课考试时间：6 月 24-25 日

### 2.2 各科提分策略

| 科目 | 提分特点 | AI老师策略 |
|------|---------|-----------|
| 数学 | 模块化强，差距大，提分空间大 | 题库诊断 + 精确薄弱点定位 + 每日推题 |
| 英语 | 听说 40 分模式固定可突破 | 听说模拟 + 阅读专项 |
| 物理 | 公式+实验，有规律可循 | 公式体系 + 实验操作要点 |
| 道法 | 开卷，答题框架 > 死记硬背 | 框架整理 + 材料分析方法 |
| 语文 | 高分段提分难，作文主观 | 维持为主，作文素材积累 |
| 体育 | 体能训练，非知识类 | 不配专属老师，班主任简单建议 |
| 考查科目 | 过等级即可，非重点 | 班主任兼任，策略性应对 |

## 三、用户与场景

### 3.1 目标用户

| 用户 | 画像 | 核心诉求 |
|------|------|---------|
| **主用户：家长** | 初三学生父母，焦虑但不知如何帮 | "孩子能上什么学校？怎么补？" |
| **执行用户：学生** | 初三学生，自驱或被安排 | "今天具体做什么？" |

> **设计优先级：家长 > 学生。** 家长是决策者、付费者、传播者。

### 3.2 使用场景

- 家长：晚上刷手机时收到班主任的学习报告，随手回复
- 学生：早上在群里看到今日任务，放学后发题目照片问老师
- 老师（Bot）：主动在关键时间节点推送，被动随时回答问题

## 四、产品形态

### 4.1 核心形态：微信私人教研组

每个学生一个独立微信群，命名为**「[学生姓名]私人教研组」**。

**群成员**：
- 家长（必须）
- 学生本人（可选加入）
- 班主任 Bot（核心角色）
- 语文老师 Bot
- 数学老师 Bot
- 英语老师 Bot
- 物理老师 Bot
- 道德与法治老师 Bot

**群内体验示例**：
```
[班主任李老师]: 子涵今天的数学任务——函数基础，45分钟，
               加油！做完可以发给数学王老师看看。

[学生]: [发送一张题目照片]

[数学王老师]: 这道题考的是一次函数图象，关键是
             理解斜率k的正负决定倾斜方向……

[家长]: 孩子今天状态不太好，能不能少一点任务？

[班主任李老师]: 没问题，今天减半，保持状态比数量更重要。
               明天补回来就好。
```

### 4.2 MVP 形态：1v1 私聊（班主任）

MVP 阶段因技术约束（iLink API 暂不支持自动建群），采用家长与班主任 1v1 私聊：

```
家长 ←→ 班主任（单一对话窗口）

班主任协调背后各科老师，发言时注明来源：
[班主任]: 我跟数学王老师确认了——这道题的解法是……
```

MVP 完成产品价值验证后，升级为完整群组形态。

### 4.3 两阶段路线图

| 阶段 | 形态 | 目标 |
|------|------|------|
| **MVP** | 1v1 私聊（班主任） | 验证对话式建档 + 主动推送 + 学科答疑的核心价值 |
| **正式版** | 微信群组（多老师） | 完整教研组体验，多老师并行 |

## 五、角色设计

### 5.1 班主任（Orchestrator）

班主任是整个系统的核心，承担以下职责：

**路由职责**：
- 所有消息默认由班主任接收
- 用户未 @特定老师时，班主任判断并指派给对应科目老师
- 考查科目（历史/化学/生物/地理）由班主任直接回答

**管理职责**：
- 维护和更新学生档案
- 主动发起对话收集信息（建档期间）
- 监控学习进度，发现异常主动关心

**推送职责**：
- 每日学习任务推送
- 每周学习报告
- 阶段里程碑提醒（一模前、二模前、冲刺期）
- 基于错题数据的举一反三推送

### 5.2 科目老师

| 老师 | 科目 | 专长 |
|------|------|------|
| 语文老师 | 语文 | 阅读理解、作文、古诗文 |
| 数学老师 | 数学 | 七大模块诊断、题目讲解、举一反三 |
| 英语老师 | 英语 | 听说机考、阅读、完形 |
| 物理老师 | 物理 | 公式推导、实验操作、计算题 |
| 道法老师 | 道德与法治 | 开卷答题框架、材料分析 |

**科目老师响应规则**：
1. 班主任 @科目老师 → 科目老师接管并回复
2. 用户直接 @科目老师 → 科目老师直接回复
3. 科目老师回复后，班主任可补充总结或跟进

### 5.3 Bot 账号规划

| 角色 | 数量 | 说明 |
|------|------|------|
| 班主任 | 1个微信账号 | 所有学生的群共用同一个班主任账号 |
| 语文/数学/英语/物理/道法老师 | 各1个微信账号 | 共5个，所有学生的群共用 |
| **合计** | **6个** | 通过群ID隔离上下文，各学生互不干扰 |

## 六、对话流程设计

### 6.1 Onboarding：从扫码到开始服务

**Step 1 — 扫码入口**

家长扫码，跳转简单网页，填写学生姓名。

**Step 2 — 引导建群**（MVP阶段手动，正式版自动）

```
系统生成引导页：

「欢迎！请按以下步骤建立[王子涵]的私人教研组

第1步：微信新建群，命名「王子涵私人教研组」
第2步：扫码添加班主任   [二维码]
第3步：扫码添加各科老师  [5个二维码]

完成后班主任会主动打招呼 👋」
```

**Step 3 — 班主任激活**

班主任进群/被加好友后，自动发出第一条消息：

```
[班主任李老师]: 王子涵家长您好！我是班主任李老师，
很高兴认识您。从今天开始，我和各科老师会在这里
陪伴王子涵备考中考 🎯

先来认识一下——王子涵现在初几了？
```

### 6.2 建档对话：逐步了解学生

班主任通过自然对话收集信息，不做问卷填表。

**建档第一阶段（前3天，了解基本情况）**：
```
班主任: 王子涵现在初几了？

家长: 初三了

班主任: 好的！距离中考还有不少时间。
       上次模考（或期末考）总分大概多少？

家长: 395分左右

班主任: 395分，有目标学校吗？比如想冲哪所高中？

家长: 想上八十中

班主任: 八十中近几年录取线在450分左右，
       差距55分，有一定挑战但完全可以冲！
       各科成绩大概是多少？我帮你看看从哪里
       补最有效率……
```

**建档第二阶段（持续进行，对话中积累）**：
- 家长发试卷照片 → 记录错题和薄弱点
- 家长提到孩子某科弱 → 更新档案优先级
- 模考成绩出来 → 更新档案，重新评估方案

### 6.3 日常：主动推送

**每日早晨任务推送**（班主任）：
```
[班主任李老师]: ☀️ 早上好！今天的学习任务：

📐 数学 · 函数基础    45分钟
   目标：一次函数图象和性质
   
🔤 英语 · 听说模拟    30分钟

今天总计75分钟，加油 💪
做完可以把数学题发给王老师看看！
```

**举一反三推送**（数学老师，事件触发）：
```
[数学王老师]: 子涵昨天这道函数题做错了，
给你出一道类似的练习一下：

[题目图片]

思路提示：先找两点，再确定斜率……
```

**每周学习报告**（班主任，每周日晚）：
```
[班主任李老师]: 📊 本周学习报告（5/4-5/10）

完成情况：计划14项，完成11项（79%）✅
数学正确率：68%，比上周提升9% 📈
学习时长：9.5小时

重点进展：
· 函数模块明显进步，基本概念清晰了
· 英语听说还需加强，建议下周多练

距中考还有45天，子涵这周很努力！
继续保持 💪
```

### 6.4 日常：被动答疑

**学生发题目照片**：
```
[学生]: [发送题目照片]

[班主任李老师]: 这是数学题，@数学王老师 来看看

[数学王老师]: 好，我看一下……
这道题是圆的切线问题，解题步骤：
1. 先连接切点和圆心
2. 切线与半径垂直，所以角度关系……
```

**家长问学校信息**：
```
[家长]: 八十中今年录取分数线多少？

[班主任李老师]: 根据近几年数据：
· 2025年：452分（海淀区）
· 2024年：448分
· 2023年：455分
子涵现在395分，差距约55分，
建议重点提数学和英语，这两科空间最大。
```

## 七、学生档案

### 7.1 档案结构

通过对话积累的核心数据：

```
学生档案（student_profile）
├── 基本信息
│   ├── 姓名、年级
│   ├── 目标学校
│   └── 所在区
├── 成绩数据
│   ├── 各科当前分数
│   ├── 总分
│   └── 历次模考记录
├── 学科画像
│   ├── 数学：7模块掌握程度
│   ├── 英语：听说/阅读/写作评估
│   └── 其他各科：薄弱点标签
├── 错题库
│   └── 按科目/知识点分类
├── 执行记录
│   ├── 每日任务完成情况
│   ├── 学习时长
│   └── 连续打卡天数
└── 对话摘要
    └── 班主任提炼的关键信息
```

### 7.2 档案完整度

档案越完整，推送越精准：

| 完整度 | 状态 | 班主任行为 |
|--------|------|-----------|
| 刚入群 | 只知道姓名 | 主动提问收集基本信息 |
| 了解基本成绩 | 可给粗略方向 | 开始推送通用任务 |
| 完成科目诊断 | 可给精确计划 | 推送个性化任务 |
| 有持续错题数据 | 高度个性化 | 精准举一反三 |

### 7.3 数据来源

| 数据 | 来源方式 |
|------|---------|
| 基本信息 | 班主任主动问 |
| 各科成绩 | 家长告知 / 模考后更新 |
| 数学薄弱点 | AI分析错题照片 |
| 错题记录 | 学生发题目照片 → Bot批改记录 |
| 执行数据 | 家长/学生在群内确认完成 |

## 八、技术架构

### 8.1 微信接入层

| 项目 | 方案 |
|------|------|
| 接入协议 | 腾讯官方 iLink Bot API（微信 ClawBot） |
| 登录方式 | 扫二维码，无需公网 IP，长轮询 |
| Bot 账号数 | 6个（班主任+5科老师） |
| 消息支持 | 文字、图片、语音、文件 |
| 当前限制 | API 暂不支持自动建群/拉人（MVP手动建群） |

### 8.2 后端架构

```
微信消息（iLink API 长轮询）
        ↓
消息路由层
  ├── 识别群ID → 找到对应学生档案
  ├── 识别@对象 → 路由给对应Bot角色
  └── 无指定 → 交给班主任Orchestrator判断
        ↓
Agent 层（多租户）
  ├── 班主任 Agent（Orchestrator）
  │     ├── 读取/更新学生档案
  │     ├── 调用各科 Agent
  │     └── 管理推送调度
  └── 科目 Agent × 5
        ├── 学科专业知识库
        └── 错题分析能力
        ↓
存储层
  ├── 学生档案 DB（多租户隔离）
  ├── 错题库
  ├── 推送调度队列
  └── 知识库（现有zhongkao-agent内容）
```

### 8.3 多租户隔离

- 同一个"数学王老师"账号服务所有学生
- 收到消息时通过**群ID**匹配对应学生档案
- 各学生数据完全隔离，互不可见

### 8.4 技术栈

| 层 | 技术 |
|----|------|
| 微信接入 | iLink Bot API（HTTP/JSON 长轮询） |
| 后端语言 | Python 或 Node.js |
| AI 模型 | 现有 MiniMax LLM + 视觉模型（图片批改） |
| 数据库 | SQLite（MVP）→ PostgreSQL（正式版） |
| 推送调度 | 定时任务（cron）+ 事件队列 |
| 知识库 | 现有 zhongkao-agent 数学题库 + 录取分数线 |

> **Web 端**：现有 Next.js 产品保留，定位为**管理后台**（查看学生档案、数据统计、运营管理），不作为用户主界面。

## 九、MVP 范围

### 9.1 MVP 目标

用最小成本验证核心假设：
> **家长愿意在微信里和 AI 班主任持续对话，并因此改善孩子的备考质量。**

### 9.2 MVP 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 1v1 私聊接入 | 班主任Bot与家长私聊 | P0 |
| 建档对话 | 通过聊天收集基本信息 | P0 |
| 成绩-学校匹配 | 告知总分后给出目标校差距 | P0 |
| 学科答疑 | 发题目照片，Bot解答 | P0 |
| 每日任务推送 | 班主任每天早上推学习任务 | P0 |
| 每周学习报告 | 班主任每周日推送总结 | P1 |
| 错题记录 | 批改后自动记录薄弱点 | P1 |
| 举一反三推送 | 基于错题生成类似题 | P1 |
| 多Bot群组 | 升级为完整教研组群聊 | P2 |
| 自动建群/拉人 | 等待 iLink API 开放 | P3 |

### 9.3 MVP Onboarding 流程

```
家长扫码 → 填写学生姓名
    ↓
系统：「请先添加班主任为好友，扫码👇」
    ↓
家长添加班主任微信
    ↓
班主任自动发第一条消息，开始建档对话
    ↓
3天内完成基本建档，开始推送日常任务
```

## 十、设计原则

1. **对话即产品**：不依赖用户主动打开 App，通过推送触达用户
2. **老师人格化**：每个 Bot 有名字、有风格，不是冷冰冰的机器人
3. **主动 > 被动**：班主任主动推送 > 等用户来问
4. **积累 > 一次性**：每次对话都在丰富档案，推送越来越精准
5. **结果承诺**：每个方案都要有量化的预期（"4周后数学预计提升8-12分"）
6. **通俗表达**：不出现技术标签，用家长听得懂的语言
7. **移动优先**：微信就是移动优先
8. **数学先行**：数学有完整题库和AI能力，优先做深
9. **紧迫感**：距中考天数始终是对话背景
10. **信任积累**：宁可少说，不说没依据的话
